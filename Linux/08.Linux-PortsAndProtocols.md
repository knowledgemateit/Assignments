## **Part 1: Understanding Ports and Protocols**

### **What is a Port?**

A port is a logical endpoint for network communication.
- Port numbers: 0 to 65535
- 2 protocols: TCP and UDP

### **Port Ranges**

| Range | Purpose |
|-------|---------|
| 0-1023 | Well-known ports (reserved) |
| 1024-49151 | Registered ports |
| 49152-65535 | Dynamic/private ports |

### **Well-Known Ports**

| Port | Protocol | Service |
|------|----------|---------|
| 20 | TCP | FTP (File Transfer Protocol) |
| 21 | TCP | FTP Control |
| 22 | TCP | SSH (Secure Shell) |
| 23 | TCP | Telnet |
| 25 | TCP | SMTP (Email) |
| 53 | TCP/UDP | DNS (Domain Name System) |
| 80 | TCP | HTTP (Web) |
| 110 | TCP | POP3 (Email) |
| 143 | TCP | IMAP (Email) |
| 443 | TCP | HTTPS (Secure Web) |
| 3306 | TCP | MySQL |
| 5432 | TCP | PostgreSQL |
| 6379 | TCP | Redis |
| 8080 | TCP | HTTP Alternate |
| 8443 | TCP | HTTPS Alternate |
| 27017 | TCP | MongoDB |

## **Part 2: Network Protocols**

### **Common Protocols**

#### **TCP (Transmission Control Protocol)**
- Connection-oriented
- Reliable delivery
- Used for: HTTP, HTTPS, SSH, FTP, SMTP, POP3, IMAP

#### **UDP (User Datagram Protocol)**
- Connectionless
- Faster but unreliable
- Used for: DNS, DHCP, SNMP, VoIP

#### **ICMP (Internet Control Message Protocol)**
- For error reporting
- Used for: ping, traceroute

### **Protocol Layers (OSI Model)**

```
Layer 7: Application (HTTP, SMTP, SSH, DNS)
Layer 6: Presentation (Encryption, Compression)
Layer 5: Session (Connection management)
Layer 4: Transport (TCP, UDP)
Layer 3: Network (IP, ICMP)
Layer 2: Data Link (MAC, Ethernet)
Layer 1: Physical (Hardware, Cables)
```

## **Part 3: Services and Daemons**

### **What is a Service/Daemon?**

A daemon is a background process that provides services.
- Typically ends with 'd' (e.g., sshd, httpd, mysqld)
- Started automatically at boot
- Managed by systemd

### **Common Services**

```bash
# Web servers
apache2 (Apache)
nginx (Nginx)

# SSH server
sshd (SSH)

# Database servers
mysqld (MySQL)
postgres (PostgreSQL)

# DNS server
named (BIND)

# Mail services
postfix (SMTP)
dovecot (IMAP/POP3)

# Network services
vsftpd (FTP)
smbd (Samba/Windows sharing)

# Caching
redis (Redis)
memcached (Memcached)
```

## **Part 4: Checking Network Services and Ports**

### **netstat Command**

```bash
# Show all listening ports
netstat -tuln

# Show all established connections
netstat -tupn

# Show statistics
netstat -s

# Show per-protocol statistics
netstat -s -t              # TCP only
netstat -s -u              # UDP only

# Watch netstat in real-time
watch netstat -tuln
```

**netstat options:**
- `-t`: TCP
- `-u`: UDP
- `-l`: Listening
- `-n`: Numeric (IP instead of hostnames)
- `-p`: Program name
- `-a`: All

### **ss Command (Modern replacement)**

```bash
# Show all listening ports
ss -tuln

# Show all connections
ss -tupn

# Show TCP connections only
ss -t

# Show UDP connections only
ss -u

# Show Unix sockets
ss -x

# Show statistics
ss -s

# Watch in real-time
watch ss -tuln
```

### **lsof Command**

```bash
# List all open files and ports
lsof

# Show processes using port
lsof -i :8080

# Show all network connections
lsof -i -P -n

# Show processes by user
lsof -u username

# Show specific process
lsof -p 1234

# Show files by user
lsof -u username
```

### **curl/nc Commands**

```bash
# Test if port is open
curl -v http://localhost:8080

# Test with telnet
telnet localhost 8080

# Test with nc
nc -zv localhost 8080

# Check multiple ports
for port in 22 80 443 3306 5432; do
  echo "Testing port $port"
  nc -zv localhost $port
done
```

## **Part 5: Firewall and Port Management**

### **ufw (Ubuntu Firewall)**

```bash
# Enable firewall
sudo ufw enable

# Disable firewall
sudo ufw disable

# Check status
sudo ufw status

# Show verbose status
sudo ufw status verbose

# Show numbered list
sudo ufw status numbered

# Allow port
sudo ufw allow 22
sudo ufw allow 80
sudo ufw allow 443

# Allow specific IP
sudo ufw allow from 192.168.1.100 to any port 22

# Deny port
sudo ufw deny 23

# Delete rule
sudo ufw delete allow 80

# Allow service
sudo ufw allow ssh
sudo ufw allow http
sudo ufw allow https

# Default policies
sudo ufw default allow incoming
sudo ufw default deny incoming
sudo ufw default deny outgoing
```

### **iptables (Manual firewall)**

```bash
# List rules
sudo iptables -L -n

# List with line numbers
sudo iptables -L -n --line-numbers

# Add rule (allow port)
sudo iptables -A INPUT -p tcp --dport 8080 -j ACCEPT

# Add rule (deny port)
sudo iptables -A INPUT -p tcp --dport 23 -j DROP

# Delete rule
sudo iptables -D INPUT 1

# Save rules (must install iptables-persistent)
sudo iptables-save > /etc/iptables/rules.v4
```

## **Part 6: Systemd Service Management**

### **Service Commands**

```bash
# Start service
sudo systemctl start apache2

# Stop service
sudo systemctl stop apache2

# Restart service
sudo systemctl restart apache2

# Reload configuration (without stopping)
sudo systemctl reload apache2

# Check status
sudo systemctl status apache2

# Enable at boot
sudo systemctl enable apache2

# Disable at boot
sudo systemctl disable apache2

# Check if enabled
sudo systemctl is-enabled apache2

# Check if running
sudo systemctl is-active apache2
```

### **View Service Logs**

```bash
# View service logs
journalctl -u apache2

# View recent logs
journalctl -u apache2 -n 20

# Follow logs in real-time
journalctl -u apache2 -f

# View logs since time
journalctl -u apache2 --since "1 hour ago"
```

## **Part 7: Opening and Closing Ports**

### **Start a Service on Port**

```bash
# Start SSH on port 22 (default)
sudo systemctl start ssh

# Start Apache on port 80 (default)
sudo systemctl start apache2

# Allow through firewall
sudo ufw allow 80

# Verify it's listening
sudo ss -tuln | grep :80
```

### **Close/Stop a Service**

```bash
# Stop service
sudo systemctl stop apache2

# Deny through firewall
sudo ufw deny 80

# Verify port is closed
sudo ss -tuln | grep :80 || echo "Port closed"
```

### **Redirect Ports**

```bash
# Redirect port 8080 to 80
sudo iptables -t nat -A PREROUTING -p tcp --dport 8080 -j REDIRECT --to-port 80

# Redirect from non-root user port to root port
sudo iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 8080
```
