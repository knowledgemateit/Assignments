## **Part 1: Network Basics**

### **OSI Network Model**

```
Layer 7: Application (HTTP, SSH, DNS)
Layer 6: Presentation
Layer 5: Session
Layer 4: Transport (TCP, UDP) ← Ports
Layer 3: Network (IP) ← IP Addresses, Routing
Layer 2: Data Link (MAC) ← Network cards
Layer 1: Physical ← Cables, Hardware
```

### **IP Addressing**

**IPv4:**
- 32-bit address (4 octets)
- Format: 192.168.1.100
- 5 classes (A, B, C, D, E)

**Common Private Ranges:**
- 10.0.0.0 to 10.255.255.255
- 172.16.0.0 to 172.31.255.255
- 192.168.0.0 to 192.168.255.255

**Subnet Mask:**
- Determines network and host portions
- Example: /24 = 255.255.255.0

## **Part 2: Network Interface Configuration**

### **View Network Interfaces**

```bash
# Show network interfaces
ip link show

# Show IP addresses
ip addr show

# Show routes
ip route show

# Show only IPv4
ip -4 addr show

# Show specific interface
ip addr show eth0

# Traditional ifconfig (may not be installed)
ifconfig
```

### **Temporary IP Configuration**

```bash
# Assign IP address
sudo ip addr add 192.168.1.100/24 dev eth0

# Remove IP address
sudo ip addr del 192.168.1.100/24 dev eth0

# Set default gateway
sudo ip route add default via 192.168.1.1

# Add route to network
sudo ip route add 192.168.2.0/24 via 192.168.1.254

# Bring interface up
sudo ip link set eth0 up

# Bring interface down
sudo ip link set eth0 down

# Change MAC address (spoofing)
sudo ip link set eth0 address 00:11:22:33:44:55
```

### **Permanent Configuration (Netplan)**

Modern Ubuntu uses Netplan. Edit YAML files in `/etc/netplan/`:

```bash
# View netplan configuration
cat /etc/netplan/00-installer-config.yaml

# Example YAML configuration
network:
  version: 2
  ethernets:
    eth0:
      dhcp4: true
    eth1:
      dhcp4: false
      addresses:
        - 192.168.1.100/24
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]

# Apply configuration
sudo netplan apply

# Test configuration
sudo netplan try

# Debug configuration
sudo netplan generate
```

### **Traditional Configuration (/etc/network/interfaces)**

Older Ubuntu/Debian systems:

```bash
# Edit interfaces file
sudo nano /etc/network/interfaces

# Example content:
# auto eth0
# iface eth0 inet dhcp
#
# auto eth1
# iface eth1 inet static
#   address 192.168.1.100
#   netmask 255.255.255.0
#   gateway 192.168.1.1

# Restart networking
sudo systemctl restart networking
```

## **Part 3: DNS Configuration**

### **Check Current DNS**

```bash
# View resolv.conf
cat /etc/resolv.conf

# Check DNS resolution
nslookup google.com
dig google.com
host google.com

# Reverse DNS lookup
nslookup 8.8.8.8
```

### **Configure DNS (systemd-resolved)**

```bash
# View current DNS status
systemd-resolve --status

# Temporary DNS change
sudo systemctl restart systemd-resolved

# Edit resolved config
sudo nano /etc/systemd/resolved.conf

# Example:
# [Resolve]
# DNS=8.8.8.8 8.8.4.4
# FallbackDNS=1.1.1.1 1.0.0.1

# Apply changes
sudo systemctl restart systemd-resolved

# Verify
systemd-resolve --status
```

### **DNS Troubleshooting**

```bash
# Test DNS resolution
nslookup example.com

# Detailed DNS query
dig example.com

# DNS trace
dig +trace example.com

# Check specific NS
dig @8.8.8.8 example.com

# Reverse lookup
dig -x 142.250.185.46
```

## **Part 4: DHCP Configuration**

DHCP automatically assigns IP addresses.

```bash
# Check DHCP status
sudo systemctl status isc-dhcp-server

# DHCP server configuration
sudo nano /etc/dhcp/dhcpd.conf

# Example configuration:
# subnet 192.168.1.0 netmask 255.255.255.0 {
#   range 192.168.1.10 192.168.1.50;
#   option domain-name-servers 8.8.8.8;
#   option routers 192.168.1.1;
#   default-lease-time 600;
#   max-lease-time 7200;
# }

# Restart DHCP server
sudo systemctl restart isc-dhcp-server

# Check DHCP leases
cat /var/lib/dhcp/dhcpd.leases
```

## **Part 5: Network Diagnostics**

### **ping Command**

```bash
# Test connectivity
ping -c 4 8.8.8.8

# Continuous ping
ping 8.8.8.8

# Specify count
ping -c 5 google.com

# Specify timeout
ping -w 5000 google.com

# Specify packet size
ping -s 1472 google.com
```

### **traceroute Command**

```bash
# Trace route to host
traceroute google.com

# Use TCP instead of UDP
traceroute -T google.com

# Specify maximum hops
traceroute -m 20 google.com

# Use IPv6
traceroute6 google.com
```

### **mtr Command (ping + traceroute)**

```bash
# Monitor route to host
mtr google.com

# Run for 10 seconds then exit
mtr -c 10 google.com

# Use TCP
mtr -T google.com
```

### **nslookup and dig**

```bash
# DNS lookup
nslookup google.com

# Specify DNS server
nslookup google.com 8.8.8.8

# Detailed DNS query
dig google.com

# Query specific record
dig google.com MX              # Mail servers
dig google.com NS              # Name servers
dig google.com A               # IPv4 address
dig google.com AAAA            # IPv6 address

# Trace DNS recursion
dig +trace google.com
```

## **Part 6: Network Monitoring**

### **netstat and ss**

```bash
# Show listening ports
netstat -tuln

# Show all connections
ss -tupn

# Show connection statistics
netstat -s

# Show network interfaces
ifconfig
ip link show
```

### **iftop Command**

```bash
# Monitor network bandwidth
sudo iftop

# Monitor specific interface
sudo iftop -i eth0

# Show source/destination
sudo iftop -B
```

### **nethogs Command**

```bash
# Show bandwidth by process
sudo nethogs

# Monitor specific interface
sudo nethogs eth0

# Exit with 'q'
```

### **tcpdump Command**

```bash
# Capture all traffic
sudo tcpdump

# Capture on specific interface
sudo tcpdump -i eth0

# Capture specific port
sudo tcpdump -i eth0 port 80

# Capture specific protocol
sudo tcpdump -i eth0 tcp

# Save to file
sudo tcpdump -i eth0 -w capture.pcap

# Read pcap file
sudo tcpdump -r capture.pcap

# Display human-readable format
sudo tcpdump -i eth0 -A
```

## **Part 7: Network Security**

### **SSH Connection**

```bash
# Connect via SSH
ssh user@hostname

# Connect with specific port
ssh -p 2222 user@hostname

# Run command via SSH
ssh user@hostname "ls /home"

# Copy files (SCP)
scp file.txt user@hostname:/remote/path/
scp user@hostname:/remote/file.txt ./

# SSH key-based authentication
ssh-keygen -t rsa -b 4096
ssh-copy-id user@hostname
ssh user@hostname                  # No password needed
```

### **Firewall with UFW**

```bash
# Enable firewall
sudo ufw enable

# Allow port
sudo ufw allow 22

# Allow service
sudo ufw allow ssh

# Deny port
sudo ufw deny 23

# Status
sudo ufw status
```

## **Part 8: Network Troubleshooting Guide**

### **Diagnose Connection Issues**

```bash
# Step 1: Check local interface
ip link show
ip addr show

# Step 2: Test local connectivity
ping 127.0.0.1                    # Loopback
ping 192.168.1.1                  # Gateway

# Step 3: Test internet connectivity
ping 8.8.8.8

# Step 4: Check DNS
nslookup google.com

# Step 5: Check routing
ip route show
traceroute google.com

# Step 6: Check open ports
ss -tuln

# Step 7: Check logs
journalctl -n 50
```

### **Common Issues and Solutions**

```bash
# No internet connection
# Solution: Check IP config, DNS, gateway
ip addr show
cat /etc/resolv.conf
ip route show

# Cannot reach specific host
# Solution: Check route, firewall
traceroute hostname
sudo ufw status

# DNS not working
# Solution: Check resolv.conf, DNS servers
cat /etc/resolv.conf
systemd-resolve --status

# Slow network
# Solution: Check bandwidth usage, connections
sudo iftop
netstat -tupn | wc -l
```
