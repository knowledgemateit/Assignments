## **Part 1: What is Linux?**

### **History and Overview**

Linux is a free, open-source operating system kernel created by **Linus Torvalds** in 1991. Key points:

- **Kernel**: Core component managing hardware resources
- **Open Source**: Source code is publicly available
- **Free**: Available at no cost
- **Portable**: Runs on servers, desktops, embedded systems, supercomputers
- **Multitasking**: Run multiple programs simultaneously
- **Multi-user**: Multiple users can work on same system

### **Linux vs Unix**

| Feature | Linux | Unix |
|---------|-------|------|
| Source | Open Source | Proprietary |
| Cost | Free | Licensed |
| Creator | Linus Torvalds | Bell Labs |
| Portability | Highly portable | Less portable |
| License | GPL | Various |

## **Part 2: Linux Distributions**

### **What is a Distribution?**

A Linux distribution (distro) is Linux kernel + GNU utilities + software management tools.

### **Major Distributions**

#### **Red Hat Based:**
- **Red Hat Enterprise Linux (RHEL)**: Commercial enterprise solution
- **CentOS**: Free, community-supported
- **Fedora**: Cutting-edge features
- **Rocky Linux**: RHEL compatible

#### **Debian Based:**
- **Debian**: Stable, rock-solid foundation
- **Ubuntu**: User-friendly, desktop and server
- **Linux Mint**: Ubuntu-based, easy for beginners
- **Kali Linux**: Security testing

#### **Others:**
- **Arch Linux**: Rolling release, minimalist
- **Alpine Linux**: Lightweight, container-friendly
- **openSUSE**: Enterprise and community versions
- **Slackware**: Simplistic, Unix-like

### **Choosing a Distribution**

**For Beginners**: Ubuntu, Linux Mint
**For Servers**: RHEL, CentOS, Debian
**For Development**: Fedora, Ubuntu
**For Security**: Kali Linux
**For Embedded**: Alpine Linux

## **Part 3: Install Linux**

### **Option 1: Ubuntu on Windows (WSL2)**

1. Open PowerShell as Administrator:
   ```powershell
   wsl --install Ubuntu
   ```

2. Wait for installation to complete

3. Launch Ubuntu:
   ```powershell
   wsl -d Ubuntu
   ```

### **Option 2: Virtual Machine**

1. Download VirtualBox: **https://www.virtualbox.org**
2. Download Ubuntu ISO: **https://ubuntu.com/download**
3. Create new VM with:
   - RAM: 2-4 GB
   - Storage: 20-30 GB
   - Network: Bridged Adapter

### **Option 3: Docker Container**

1. Run Linux container:
   ```powershell
   docker run -it ubuntu:22.04 bash
   ```

### **Option 4: Cloud Linux**

- AWS EC2
- Azure Virtual Machines
- Google Cloud Compute
- DigitalOcean

## **Part 6: Linux Philosophy**

### **Core Principles**

1. **Everything is a file**
   - Programs, devices, directories = files

2. **Small is beautiful**
   - Focused, single-purpose tools

3. **Do one thing well**
   - Master one function before adding more

4. **Make programs work together**
   - Pipe output between programs

5. **Use plain text**
   - Human-readable configuration

6. **Leverage existing tools**
   - Reuse instead of recreate

### **Examples**

```bash
# Everything is a file
cat /proc/cpuinfo        # CPU info as file
cat /proc/meminfo        # Memory info as file
cat /dev/zero            # Zero device

# Combine tools (pipes)
ls | grep .txt           # List txt files
cat file.txt | sort      # Sort file contents
cat file.txt | wc -l     # Count lines
```

## **Part 7: Linux File Types**

Linux supports various file types:

| Type | Symbol | Description |
|------|--------|-------------|
| Regular File | `-` | Text, binary, image files |
| Directory | `d` | Container for files |
| Link | `l` | Shortcut to another file |
| Device | `c`, `b` | Hardware devices |
| Socket | `s` | Network communication |
| Pipe | `p` | Inter-process communication |

### **Check File Type**

```bash
# ls -l shows file type (first character)
ls -l

# file command identifies file type
file /etc/passwd
file /bin/bash
file image.jpg
```

## **Part 8: Linux Boot Process**

### **5 Stages of Linux Boot**

1. **BIOS/UEFI**
   - Hardware initialization
   - POST (Power-On Self Test)

2. **Bootloader**
   - GRUB loads kernel
   - Kernel parameters passed

3. **Kernel Initialization**
   - Hardware detected
   - Root filesystem mounted

4. **Init Process**
   - systemd starts (modern Linux)
   - Services initialized

5. **Shell Prompt**
   - Login prompt appears
   - User can interact

### **View Boot Messages**

```bash
# View boot log
dmesg | head -20

# View systemd boot time
systemd-analyze
```

## **Part 9: Getting Help**

### **Help Resources**

```bash
# Manual pages (most comprehensive)
man ls
man grep
man bash

# Command help
ls --help
grep -h

# Search manual pages
man -k search_term

# Info pages (GNU documentation)
info bash

# Online help
# Visit man7.org or linux.die.net
```

### **Manual Page Sections**

```
1 - User commands
2 - System calls
3 - C library functions
4 - Device files
5 - Configuration files
6 - Games
7 - Miscellaneous
8 - Administration commands
9 - Kernel internals
```

### **Navigate Manual Pages**

- `Space`: Next page
- `b`: Previous page
- `/search`: Search
- `n`: Next search result
- `q`: Quit
