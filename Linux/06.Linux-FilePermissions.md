## **Part 1: Permission Basics**

### **Three Permission Types**

| Permission | Symbol | Number | Description |
|-----------|--------|--------|-------------|
| Read | r | 4 | View file contents |
| Write | w | 2 | Modify file contents |
| Execute | x | 1 | Run file (for programs/scripts) |

For directories:
- **Read (r)**: List directory contents
- **Write (w)**: Create/delete files in directory
- **Execute (x)**: Enter/access directory

### **Three User Categories**

| Category | Symbol | Description |
|----------|--------|-------------|
| Owner | u | File/folder owner |
| Group | g | Users in file's group |
| Other | o | All other users |
| All | a | Everyone |

### **Permission Notation**

```bash
# Example: -rw-r--r--
# - : Regular file
# rw- : Owner can read and write (no execute)
# r-- : Group can only read
# r-- : Others can only read

# Example: drwxr-xr-x
# d : Directory
# rwx : Owner can read, write, enter
# r-x : Group can read and enter
# r-x : Others can read and enter
```

## **Part 2: Viewing Permissions**

### **List with Permissions**

```bash
# View file permissions
ls -l myfile.txt

# View directory permissions
ls -ld mydir/

# View all files with permissions
ls -la

# View recursive permissions
ls -lR

# Show permissions in octal
stat myfile.txt
```

### **Understanding Permission Output**

```bash
# Example output: -rw-r--r-- 1 user group 1234 Jan 15 10:30 myfile.txt

# Breaking it down:
# [-rw-r--r--] [1] [user] [group] [1234] [Date] [filename]
#  File Type + Permissions | Links | Owner | Group | Size | Modified | Name
```

## **Part 3: Changing Permissions (chmod)**

### **Symbolic Mode**

```bash
# Add execute permission for owner
chmod u+x myfile.txt

# Remove write permission for group
chmod g-w myfile.txt

# Add read/write for everyone
chmod a+rw myfile.txt

# Remove all permissions
chmod a-rwx myfile.txt

# Set specific permissions
chmod u=rwx,g=rx,o=r myfile.txt

# Apply to directory recursively
chmod -R u+rwx,g+rx,o+rx mydir/

# Verbose output
chmod -v u+x myfile.txt
```

### **Numeric Mode (Octal)**

```bash
# Permission values:
# Read (r) = 4
# Write (w) = 2
# Execute (x) = 1

# Examples:
chmod 755 myfile.txt        # rwxr-xr-x (7+5+5)
chmod 644 myfile.txt        # rw-r--r-- (6+4+4)
chmod 755 mydir/            # rwxr-xr-x
chmod 700 myfile.txt        # rwx------ (7+0+0)
chmod 777 myfile.txt        # rwxrwxrwx (everyone full access)
chmod 600 myfile.txt        # rw------- (owner only)
chmod 777 -R mydir/         # Recursive chmod

# Common permission values:
# 644 (rw-r--r--) - Regular files
# 755 (rwxr-xr-x) - Executable files
# 700 (rwx------) - Private files
# 777 (rwxrwxrwx) - Full access (rarely used)
```

### **Changing Default Permissions (umask)**

```bash
# View current umask
umask

# umask determines which permissions are REMOVED from defaults
# Default file permissions: 666 (rw-rw-rw-)
# Default directory permissions: 777 (rwxrwxrwx)

# Set umask (temporarily)
umask 0022

# Permanent umask (add to ~/.bashrc)
echo "umask 0022" >> ~/.bashrc

# Common umask values:
# 0022 = 644 for files, 755 for directories
# 0077 = 600 for files, 700 for directories
# 0027 = 640 for files, 750 for directories

# Calculate permissions:
# File: 666 - umask(022) = 644 (rw-r--r--)
# Dir: 777 - umask(022) = 755 (rwxr-xr-x)
```

## **Part 4: Changing Ownership**

### **chown Command (change owner)**

```bash
# Change file owner
sudo chown newuser myfile.txt

# Change owner and group
sudo chown newuser:newgroup myfile.txt

# Change group only
sudo chown :newgroup myfile.txt

# Change recursively
sudo chown -R newuser:newgroup mydir/

# Verbose output
sudo chown -v newuser myfile.txt

# Copy ownership from another file
sudo chown --reference=source.txt target.txt
```

### **chgrp Command (change group)**

```bash
# Change group of file
sudo chgrp newgroup myfile.txt

# Change recursively
sudo chgrp -R newgroup mydir/

# Verbose output
sudo chgrp -v newgroup myfile.txt
```

## **Part 5: Special Permissions**

### **SUID (Set User ID)**

When set on executable file, runs with owner's permissions.

```bash
# View SUID permission
ls -l /usr/bin/sudo         # Should show 's' instead of 'x'

# Set SUID
sudo chmod u+s myprogram

# Remove SUID
sudo chmod u-s myprogram

# Octal notation (4xxx)
sudo chmod 4755 myprogram
```

### **SGID (Set Group ID)**

When set on file, runs with group's permissions.
When set on directory, files inherit directory's group.

```bash
# View SGID
ls -l /usr/bin/wall         # Shows 's' for group

# Set SGID on file
sudo chmod g+s myprogram

# Set SGID on directory
sudo chmod g+s mydir/

# Files created in directory inherit group
sudo chmod g+s /shared-folder/

# Octal notation (2xxx)
sudo chmod 2755 myprogram
sudo chmod 2755 mydir/
```

### **Sticky Bit**

When set on directory, only owner can delete own files (even if others have write).

```bash
# View sticky bit
ls -ld /tmp                 # Shows 't' instead of 'x'

# Set sticky bit
sudo chmod +t mydir/

# Remove sticky bit
sudo chmod -t mydir/

# Octal notation (1xxx)
sudo chmod 1777 mydir/

# Common: /tmp is world-writable but sticky
sudo chmod 1777 /tmp
```

## **Part 6: File Attributes**

### **Extended Attributes**

```bash
# View extended attributes
lsattr myfile.txt

# Make file immutable
sudo chattr +i myfile.txt

# Make file append-only
sudo chattr +a myfile.txt

# Remove immutable flag
sudo chattr -i myfile.txt

# Remove append-only flag
sudo chattr -a myfile.txt

# View only immutable files
lsattr -a | grep '^\-'
```

## **Part 7: ACL (Access Control Lists)**

### **Extended Permissions**

```bash
# View ACL
getfacl myfile.txt

# Set ACL for specific user
sudo setfacl -m u:username:rwx myfile.txt

# Set ACL for specific group
sudo setfacl -m g:groupname:rwx myfile.txt

# Remove ACL entry
sudo setfacl -x u:username myfile.txt

# Remove all ACL
sudo setfacl -b myfile.txt

# Apply recursively
sudo setfacl -R -m u:username:rwx mydir/
```

## **Part 9: Permission Troubleshooting**

### **Common Issues**

```bash
# Cannot read file
# Solution: Check owner has read permission
chmod u+r myfile.txt

# Cannot execute script
# Solution: Add execute permission
chmod u+x myscript.sh

# Cannot write to directory
# Solution: Check write permission on directory
chmod u+w mydir/

# Cannot access subdirectory
# Solution: Directory needs execute permission
chmod u+x mydir/

# Permission denied for everyone
# Solution: Set appropriate group/other permissions
chmod g+r,o+r myfile.txt
```
