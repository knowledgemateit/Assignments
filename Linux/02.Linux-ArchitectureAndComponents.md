## **Part 1: Linux Architecture Overview**

### **Layered Architecture**

Linux follows a layered architecture:

```
┌─────────────────────────────────────┐
│     Applications/User Programs      │  ← User Space
├─────────────────────────────────────┤
│    Shells, Utilities, Libraries     │  ← User Space
├─────────────────────────────────────┤
│          System Call Interface      │  ← Boundary
├─────────────────────────────────────┤
│  Kernel (Process/Memory/FileSystem) │  ← Kernel Space
├─────────────────────────────────────┤
│       Hardware & Device Drivers     │  ← Hardware Layer
└─────────────────────────────────────┘
```

## **Part 2: Kernel Architecture**

### **Linux Kernel Components**

#### **1. Process Manager**
- Creates and manages processes
- Process scheduling
- Interprocess communication (IPC)

#### **2. Memory Manager**
- Virtual memory
- Paging and swapping
- Cache management

#### **3. File System Manager**
- File and directory management
- File operations (read, write, delete)
- File permissions

#### **4. Device Manager**
- Device driver management
- Hardware communication
- I/O operations

#### **5. Network Manager**
- Network protocols (TCP/IP)
- Socket management
- Routing and firewall

### **View Kernel Information**

```bash
# Show kernel version
uname -r

# Show all kernel info
uname -a

# Show kernel in detail
cat /proc/version

# Show kernel boot parameters
cat /proc/cmdline

# View kernel modules
lsmod

# View detailed kernel information
cat /proc/meminfo
cat /proc/cpuinfo
```

## **Part 3: User Space vs Kernel Space**

### **Key Differences**

| Aspect | Kernel Space | User Space |
|--------|--------------|-----------|
| Privilege | Full hardware access | Limited access |
| Memory | Direct memory access | Protected memory |
| Speed | Fast | Slower |
| Crashes | System crash | Process crash |
| Examples | Kernel, drivers | Applications, shells |

### **System Calls Bridge Gap**

```bash
# View available system calls
man 2 intro

# Example system calls
strace ls          # Trace system calls
```

### **Running a Command - System Call Flow**

```
User Program → Shell → System Call → Kernel → Hardware → Result
```

## **Part 4: Linux Kernel Structure**

### **Monolithic vs Modular Kernel**

Linux uses **monolithic modular kernel**:
- Monolithic: All components in single block
- Modular: Components can be loaded/unloaded

```bash
# View loaded kernel modules
lsmod

# Check if module is loaded
lsmod | grep module_name

# Load a module
sudo modprobe module_name

# Unload a module
sudo rmmod module_name

# View module information
modinfo module_name
```

### **Kernel Configuration**

```bash
# View kernel config (if available)
cat /boot/config-$(uname -r)

# Show enabled features
grep CONFIG_* /boot/config-$(uname -r) | grep yes
```

## **Part 5: System Components**

### **1. Bootloader (GRUB)**

GRUB (Grand Unified Bootloader) loads kernel and initiates boot.

```bash
# View GRUB configuration
cat /etc/default/grub

# View GRUB menu configuration
cat /boot/grub/grub.cfg
```

### **2. Init System (systemd)**

Modern Linux uses **systemd** for system and service management.

```bash
# Check if systemd is running
ps aux | grep systemd

# View systemd version
systemctl --version

# List all services
systemctl list-unit-files

# View service status
systemctl status ssh
```

### **3. Shell**

Shell is the command interpreter.

```bash
# Check current shell
echo $SHELL

# List available shells
cat /etc/shells

# Switch shell
chsh -s /bin/bash

# View bash version
bash --version
```

### **4. System Utilities**

```bash
# File management utilities
ls, cp, mv, rm

# Text processing utilities
grep, sed, awk, cat

# System utilities
ps, top, df, du, free

# Network utilities
ping, ifconfig, netstat, ss
```

## **Part 6: Device Drivers**

### **What are Device Drivers?**

Device drivers are software that allow kernel to communicate with hardware.

```bash
# List PCI devices
lspci

# List USB devices
lsusb

# List block devices
lsblk

# View device tree
tree /dev/ 2>/dev/null | head -30

# View all devices
ls -la /dev/
```

### **Device Types**

#### **1. Character Devices**
- Stream of data (keyboard, mouse)
- Access sequentially

```bash
# Examples
ls -l /dev/tty*
ls -l /dev/null
```

#### **2. Block Devices**
- Random access (hard disk, USB)
- Access by blocks

```bash
# Examples
ls -l /dev/sd*
ls -l /dev/nvme*
```

#### **3. Network Devices**
- Network communication
- eth0, wlan0, lo

```bash
# List network interfaces
ip link show
```

## **Part 7: Process and Thread Management**

### **Processes in Linux**

```bash
# View all processes
ps aux

# View process tree
pstree

# View process details
ps -ef

# Show specific process
ps aux | grep ssh
```

### **Process States**

```
R (Running) → Actively executing
S (Sleeping) → Waiting for event
D (Uninterruptible) → Waiting for I/O
Z (Zombie) → Terminated but not cleaned
T (Stopped) → Suspended
```

### **View Process Information**

```bash
# Show process hierarchy
pstree -p

# Show process with full details
ps -eo pid,ppid,cmd,etime

# Monitor processes in real-time
top
htop
```
