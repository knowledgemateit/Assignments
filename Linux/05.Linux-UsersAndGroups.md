## **Part 1: User and Group Concepts**

### **What is a User?**

A user is an individual account with:
- Username (login name)
- User ID (UID)
- Home directory
- Login shell
- Password

### **What is a Group?**

A group is a collection of users with:
- Group name
- Group ID (GID)
- Members

### **System vs Regular Users**

```bash
# System users (UID < 1000)
# - For system services
# - No login shell typically

# Regular users (UID >= 1000)
# - For human users
# - Can login to system
```

## **Part 2: Understanding User and Group Files**

### **User Database File (/etc/passwd)**

```bash
# View user database
cat /etc/passwd

# Format: username:password:uid:gid:gecos:home:shell
# Example: user:x:1000:1000:User Name:/home/user:/bin/bash
```

**Fields:**
1. **Username**: Login name
2. **Password**: 'x' (encrypted in /etc/shadow)
3. **UID**: User ID number
4. **GID**: Primary group ID
5. **GECOS**: Full name, phone, etc.
6. **Home**: Home directory path
7. **Shell**: Login shell

```bash
# View specific user
grep username /etc/passwd

# Count users
wc -l /etc/passwd

# View system users only (UID < 1000)
awk -F: '$3 < 1000' /etc/passwd

# View regular users only (UID >= 1000)
awk -F: '$3 >= 1000' /etc/passwd
```

### **Group Database File (/etc/group)**

```bash
# View group database
cat /etc/group

# Format: groupname:password:gid:members
# Example: group:x:1000:user1,user2,user3
```

**Fields:**
1. **Groupname**: Group name
2. **Password**: Rarely used (x or blank)
3. **GID**: Group ID number
4. **Members**: Comma-separated usernames

```bash
# View specific group
grep groupname /etc/group

# View user's groups
groups username
```

### **Shadow Files (/etc/shadow and /etc/gshadow)**

```bash
# View shadow file (requires sudo)
sudo cat /etc/shadow

# Format: username:password_hash:lastchange:min:max:warn:inactive:expire

# View group shadow
sudo cat /etc/gshadow
```

## **Part 3: Get User Information**

### **Current User Information**

```bash
# Show current user
whoami

# Show current user ID and groups
id

# Show user groups
groups

# Show detailed user info
finger username          # May not be installed

# Show login information
w                        # Show who's logged in
who                      # Simple login list
```

### **View All Users**

```bash
# List all users
cat /etc/passwd

# Count total users
wc -l /etc/passwd

# List in nice format
cat /etc/passwd | cut -d: -f1

# Show only regular users
awk -F: '$3 >= 1000' /etc/passwd | cut -d: -f1
```

### **View All Groups**

```bash
# List all groups
cat /etc/group

# Count groups
wc -l /etc/group

# List group names
cat /etc/group | cut -d: -f1

# Show members of group
grep groupname /etc/group | cut -d: -f4
```

## **Part 4: Create and Manage Users**

### **Create User (useradd)**

```bash
# Basic user creation (requires sudo)
sudo useradd username

# Create with home directory
sudo useradd -m username

# Create with specific home directory
sudo useradd -m -d /home/custom username

# Create with specific UID
sudo useradd -m -u 2000 username

# Create with specific group
sudo useradd -m -g groupname username

# Create with specific shell
sudo useradd -m -s /bin/bash username

# Create with full name
sudo useradd -m -c "Full Name" username

# Create inactive user (expires on date)
sudo useradd -m -e 2024-12-31 username

# Create with combined options
sudo useradd -m -u 2000 -s /bin/bash -c "Test User" testuser
```

### **Modify User (usermod)**

```bash
# Change home directory
sudo usermod -d /home/newhome username

# Change login shell
sudo usermod -s /bin/zsh username

# Add to supplementary group
sudo usermod -a -G groupname username

# Change UID
sudo usermod -u 2001 username

# Change primary group
sudo usermod -g groupname username

# Rename user
sudo usermod -l newname oldname

# Lock user account
sudo usermod -L username

# Unlock user account
sudo usermod -U username
```

### **Delete User (userdel)**

```bash
# Delete user (keeps home directory)
sudo userdel username

# Delete user and home directory
sudo userdel -r username

# Delete user and home directory and mail
sudo userdel -r -f username
```

## **Part 5: Manage User Passwords**

### **Set Password (passwd)**

```bash
# Set password for current user
passwd

# Set password for another user (requires sudo)
sudo passwd username

# Force password change on next login
sudo passwd -e username

# Lock user account
sudo passwd -l username

# Unlock user account
sudo passwd -u username

# Set password to expire in days
sudo passwd -x 90 username

# Warn before password expires
sudo passwd -w 7 username
```

### **Generate Random Password**

```bash
# Generate random password
openssl rand -base64 12

# Set password using echo
echo "newpassword" | sudo passwd --stdin username 2>/dev/null
```

## **Part 6: Create and Manage Groups**

### **Create Group (groupadd)**

```bash
# Basic group creation (requires sudo)
sudo groupadd groupname

# Create with specific GID
sudo groupadd -g 2000 groupname

# Create system group (GID < 1000)
sudo groupadd -r systemgroup
```

### **Modify Group (groupmod)**

```bash
# Change group name
sudo groupmod -n newname oldname

# Change GID
sudo groupmod -g 2001 groupname
```

### **Delete Group (groupdel)**

```bash
# Delete group
sudo groupdel groupname

# Note: Cannot delete primary group of any user
```

### **Manage Group Members (gpasswd)**

```bash
# Add user to group
sudo gpasswd -a username groupname

# Remove user from group
sudo gpasswd -d username groupname

# Add multiple users to group
sudo gpasswd -M user1,user2,user3 groupname

# Set group administrator
sudo gpasswd -A adminuser groupname
```

## **Part 7: Sudo and Elevated Privileges**

### **Understanding sudo**

```bash
# Execute command as superuser
sudo command

# Execute as specific user
sudo -u username command

# Execute as specific user and group
sudo -u username -g groupname command

# List sudo permissions
sudo -l

# Edit sudoers file (ALWAYS use visudo)
sudo visudo

# Execute previous command with sudo
sudo !!
```

### **Sudoers File**

```bash
# Sudoers file location
/etc/sudoers

# Edit sudoers (use visudo - validates syntax)
sudo visudo

# Example entries:
# username ALL=(ALL) ALL              # Full access
# username ALL=(ALL) NOPASSWD: ALL    # No password needed
# %groupname ALL=(ALL) ALL            # Group access
# user1 ALL=(user2) /bin/ls           # Run specific command as user2
```

### **Sudoers Examples**

```bash
# Allow user to run all commands without password
sudo tee -a /etc/sudoers << EOF
username ALL=(ALL) NOPASSWD: ALL
EOF

# Allow group to run specific commands
echo "%developers ALL=(ALL) /usr/bin/systemctl" | sudo tee -a /etc/sudoers
```

## **Part 8: Practice Exercises**

### **Exercise 1: Create Users and Groups**

```bash
# 1. Create a group
sudo groupadd developers

# 2. Create a user
sudo useradd -m -s /bin/bash -c "Developer User" developer1

sudo usermod -a -G developers developer1

cat /etc/passwd | grep developer1

# 5. Verify group membership

```

### **Exercise 2: Set User Password and Permissions**

```bash
# 1. Set password for new user
sudo passwd developer1

# 2. Check password expiry settings
sudo chage -l developer1

# 3. Set password expiry
sudo chage -M 90 -W 7 developer1

# 4. Force password change on next login
sudo chage -d 0 developer1
```

### **Exercise 3: Manage Sudo Access**

```bash
# 1. Check current sudo permissions
sudo -l

# 2. Check visudo syntax
sudo visudo -c

# 3. Give user sudo access
echo "developer1 ALL=(ALL) NOPASSWD: /usr/bin/apt" | sudo tee -a /etc/sudoers

# 4. Verify sudo access for specific command
sudo -u developer1 sudo /usr/bin/apt list --installed
```

### **Exercise 4: User Login and Shell**

```bash
# 1. Switch to another user
su - developer1

# 2. Check shell
echo $SHELL

# 3. Change shell
sudo usermod -s /bin/zsh developer1

# 4. Exit user session
exit
```

## **Part 9: User and Group Best Practices**

### **Security Tips**

```bash
# 1. Strong passwords
# Use: UPPERCASE + lowercase + numbers + special chars

# 2. Regular password updates
sudo chage -M 90 username

# 3. Remove inactive users
sudo userdel -r inactiveuser

# 4. Regular audit
cat /etc/passwd | wc -l

# 5. Limit root login
sudo usermod -L root
```

### **Group Best Practices**

```bash
# 1. Create specific groups for services
sudo groupadd www-data
sudo groupadd mysql-users

# 2. Add users to appropriate groups
sudo usermod -a -G www-data username

# 3. Check group membership
grep groupname /etc/group
```
