## **Part 1: Understanding Shells**

### **What is a Shell?**

A shell is a command interpreter that reads user commands and executes them. Common shells include:
- **Bash** (Bourne Again Shell) - Most popular
- **Sh** (Bourne Shell) - POSIX standard
- **Zsh** (Z Shell) - Modern alternative
- **Ksh** (Korn Shell) - Advanced features

### **Step 1: Check Available Shells**

```bash
# List available shells
cat /etc/shells

# Check current shell
echo $SHELL

# Check default shell for user
grep username /etc/passwd
```

### **Step 2: Switch Shells (Temporary)**

```bash
# Switch to Bash
bash

# Switch to Sh
sh

# Exit current shell
exit
```

## **Part 2: Creating Your First Script**

### **Step 1: Create a Script File**

```bash
# Navigate to your working directory
cd ~/shell-scripts
mkdir -p ~/shell-scripts

# Create first script
cat > hello.sh << 'EOF'
#!/bin/bash
echo "Hello, World!"
EOF
```

### **Step 2: Add Execute Permissions**

```bash
# Add execute permission
chmod +x hello.sh

# Verify permissions
ls -l hello.sh
```

### **Step 3: Execute the Script**

```bash
# Method 1: Direct execution
./hello.sh

# Method 2: Using bash command
bash hello.sh

# Method 3: Using source command
source hello.sh
```

## **Part 3: Shebang and Script Structure**

### **Understanding Shebang (#!)**

The shebang tells the OS which interpreter to use:

```bash
#!/bin/bash       # Use Bash
#!/bin/sh         # Use Bourne Shell
#!/bin/zsh        # Use Zsh
#!/usr/bin/env bash  # Portable Bash
```

### **Step 1: Create Script with Proper Structure**

```bash
cat > script_structure.sh << 'EOF'
#!/bin/bash

# Script Header
# Title: My Script Title
# Description: What this script does
# Author: Your Name
# Date: 2024-01-01

# Variable declarations
SCRIPT_NAME="script_structure"
VERSION="1.0"

# Main script content
echo "Script: $SCRIPT_NAME"
echo "Version: $VERSION"

# Exit with status code
exit 0
EOF

chmod +x script_structure.sh
./script_structure.sh
```

## **Part 4: Comments and Documentation**

### **Step 1: Using Comments**

```bash
#!/bin/bash

# Single line comment - This is a comment
# It doesn't execute

# Multi-line comments
: '
This is a multi-line comment
Everything here is ignored
Useful for documentation
'

echo "This executes"
```

### **Step 2: Good Documentation Practice**

```bash
#!/bin/bash

##############################################################
# Script: system_info.sh
# Purpose: Display system information
# Author: Your Name
# Date: 2024-01-01
# Usage: ./system_info.sh
##############################################################

echo "System Information"
echo "=================="
echo "Hostname: $(hostname)"
echo "Uptime: $(uptime)"
```

## **Part 5: Basic Output Commands**

### **Step 1: Echo Command**

```bash
#!/bin/bash

# Simple echo
echo "Hello, World!"

# Echo with variables
name="Khajanaraju"
echo "Hello, $name"

# Echo with special characters
echo "This is a \"quoted\" string"

# Echo with escape sequences
echo -e "Line 1\nLine 2\nLine 3"

# Echo without newline
echo -n "No newline"
```

### **Step 2: Printf Command**

```bash
#!/bin/bash

# Printf with formatting
printf "Name: %s\n" "Khajanaraju"
printf "Age: %d\n" 30
printf "Salary: %.2f\n" 50000.50

# Multiple values
printf "%s - %s\n" "Item1" "Item2"
```

## **Part 6: Command-Line Arguments**

### **Step 1: Accessing Arguments**

```bash
#!/bin/bash

# $0 = script name
# $1, $2, $3... = arguments
# $@ = all arguments
# $# = number of arguments

echo "Script name: $0"
echo "First argument: $1"
echo "Second argument: $2"
echo "All arguments: $@"
echo "Total arguments: $#"
```

**Usage:**
```bash
./script.sh arg1 arg2 arg3
```

### **Step 2: Processing Arguments**

```bash
#!/bin/bash

if [ $# -eq 0 ]; then
    echo "Usage: $0 <name>"
    exit 1
fi

name=$1
echo "Hello, $name!"
```

## **Part 7: Script Execution Methods**

### **Different Ways to Execute Scripts**

```bash
# Method 1: Direct execution (requires execute permission)
./script.sh

# Method 2: Using bash
bash script.sh

# Method 3: Using sh
sh script.sh

# Method 4: Using source (runs in current shell)
source script.sh
. script.sh

# Method 5: Using full path
/home/user/scripts/script.sh
```

## **Troubleshooting Guide**

### **Common Issues**

| Issue | Solution |
|-------|----------|
| Permission denied | Use `chmod +x scriptname` |
| Command not found | Ensure shebang is correct, check PATH |
| Script not executing | Use `bash scriptname` or `./scriptname` |
| Variables not working | Check for proper syntax: `$variable` |
| Output not showing | Use `echo` command to display output |

## **Best Practices**

1. **Always use shebang** (#!) at the beginning
2. **Make scripts executable** with chmod +x
3. **Add comments** for clarity and documentation
4. **Use descriptive names** for scripts and variables
5. **Handle arguments properly** with error checking
6. **Test with different inputs** before deployment
7. **Use proper indentation** for readability
8. **Follow naming conventions** (lowercase with underscores)
9. **Document script purpose** in header comments
10. **Provide usage information** when arguments are required
