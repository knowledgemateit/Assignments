## **Part 1: Exit Codes**

### **Understanding Exit Codes**

```bash
#!/bin/bash

# Exit with success (0)
echo "Success"
exit 0

# Exit with error (non-zero)
exit 1

# Check last command's exit code
ls /tmp
if [ $? -eq 0 ]; then
    echo "Directory exists"
else
    echo "Directory not found"
fi

# Automatic exit code propagation
grep "pattern" /nonexistent/file
# $? will contain grep's exit code
```

### **Common Exit Codes**

```bash
#!/bin/bash

# Standard exit codes
exit 0   # Success
exit 1   # General error
exit 2   # Misuse of shell command
exit 126 # Command invoked cannot execute
exit 127 # Command not found
exit 128 # Invalid argument to exit
exit 130 # Script terminated by Ctrl+C

# Custom exit codes (64-113 available)
exit 64  # Custom error 1
exit 65  # Custom error 2
```

## **Part 2: Error Checking**

### **Basic Error Checking**

```bash
#!/bin/bash

# Check if file exists
file="/etc/hostname"
if [ ! -f "$file" ]; then
    echo "Error: File not found: $file" >&2
    exit 1
fi

# Check command success
if ! grep "pattern" "$file"; then
    echo "Error: Pattern not found" >&2
    exit 1
fi

# Check directory exists
if [ ! -d "/tmp" ]; then
    echo "Error: Directory not found" >&2
    exit 1
fi

# Check variable not empty
if [ -z "$username" ]; then
    echo "Error: Username required" >&2
    exit 1
fi
```

### **Set Error Options**

```bash
#!/bin/bash

# Exit on any error
set -e

# Exit on undefined variable
set -u

# Pipe fails if any command fails
set -o pipefail

# Combined
set -euo pipefail

# Disable option
set +e

# Script with safe options
#!/bin/bash
set -euo pipefail

IFS=$'\n\t'

# Now script will exit on errors
command1
command2
command3
```

## **Part 3: Try-Catch Pattern**

### **Implementing Try-Catch**

```bash
#!/bin/bash

# Function with error handling
function try() {
    [[ $- = *e* ]]; SAVED_OPT_E=$?
    set +e
}

function catch() {
    export exception_code=$?
    (( SAVED_OPT_E )) && set +e
    return $exception_code
}

# Usage
try
(
    echo "Attempting operation"
    ls /nonexistent/file
    echo "If we get here, no error"
)
catch || {
    case $exception_code in
        2)
            echo "Caught error: File not found"
            ;;
        *)
            echo "Caught error code: $exception_code"
            ;;
    esac
}
```

### **Simple Error Handler**

```bash
#!/bin/bash
```

## **Part 4: Trap and Signal Handling**

### **Using Trap**

```bash
#!/bin/bash

# Cleanup function
cleanup() {
    echo "Cleaning up..."
    rm -f /tmp/tempfile
    exit 0
}

# Set trap for signals
trap cleanup EXIT
trap cleanup INT TERM

# Script continues
echo "Script running"
sleep 100
```

### **Error Trapping**

```bash
#!/bin/bash

# Trap errors
trap 'echo "Error on line $LINENO"' ERR

# Trap with line number
error_handler() {
    local line=$1
    echo "Error occurred on line $line"
    exit 1
}

trap 'error_handler $LINENO' ERR

# This will trigger error
# undefined_variable=$nonexistent
```

### **Debug Trapping**

```bash
#!/bin/bash

# Debug mode
if [ "${DEBUG:-0}" = "1" ]; then
    set -x
    PS4='+ ${BASH_SOURCE}:${LINENO} '
fi

# Trap debug
trap 'echo "Executing: $BASH_COMMAND"' DEBUG
```

## **Part 6: Assertion and Validation**

### **Assertions**

```bash
#!/bin/bash

# Assert function
assert() {
    if [ ! $1 ]; then
        echo "ASSERTION FAILED: $2"
        exit 1
    fi
}

# Assert equal
assert_equal() {
    if [ "$1" != "$2" ]; then
        echo "ASSERTION FAILED: '$1' != '$2'"
        exit 1
    fi
}

# Usage
result=$((2 + 2))
assert_equal "$result" "4" "Addition failed"
```

### **Input Validation**

```bash
#!/bin/bash

# Validate function existence
function_exists() {
    declare -f "$1" > /dev/null
    return $?
}

# Validate command existence
command_exists() {
    command -v "$1" >/dev/null 2>&1
    return $?
}

# Validate directory
validate_directory() {
    if [ ! -d "$1" ]; then
        echo "Error: Not a directory: $1" >&2
        return 1
    fi
}

# Validate file readable
validate_readable() {
    if [ ! -r "$1" ]; then
        echo "Error: File not readable: $1" >&2
        return 1
    fi
}
```

## **Troubleshooting Guide**

| Issue | Solution |
|-------|----------|
| Errors not caught | Use `set -e` and check return codes |
| Undefined variable error | Use `set -u` to catch early |
| Pipe failures ignored | Use `set -o pipefail` |
| Trap not executing | Ensure trap is defined before triggering signal |
| IFS issues | Set IFS explicitly at script start |

## **Best Practices**

1. **Always check exit codes** of critical commands
2. **Use `set -euo pipefail`** at script start
3. **Validate all input** before use
4. **Provide meaningful error messages** to stderr
5. **Use trap for cleanup** and signal handling
6. **Support debug mode** with `-x` option
7. **Test error paths** as thoroughly as happy paths
8. **Document error codes** in comments
9. **Use meaningful variable names** in error messages
10. **Exit with appropriate codes** (0 for success, 1-255 for errors)
