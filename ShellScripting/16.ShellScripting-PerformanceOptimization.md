## **Part 1: Script Profiling**

### **Timing Script Execution**

```bash
#!/bin/bash

# Measure execution time
time ./script.sh

# Detailed timing
{ time ./script.sh; } 2>&1 | grep real

# Time specific section
start_time=$(date +%s)
# Code to measure
end_time=$(date +%s)
echo "Elapsed: $((end_time - start_time)) seconds"
```

## **Part 2: Common Performance Issues**

### **Inefficient Patterns**

```bash
# ❌ Inefficient: Spawning subshell for each iteration
for file in *.txt; do
    cat "$file" | grep "pattern"
done

# ✅ Efficient: Use grep directly
grep "pattern" *.txt

# ❌ Inefficient: Multiple file reads
for i in {1..100}; do
    grep "line$i" largefile.txt
done

# ✅ Efficient: Single pass
grep -E "line(1|2|3)" largefile.txt
```

## **Part 3: Optimization Techniques**

### **Reduce Subshell Calls**

```bash
#!/bin/bash

# ❌ Slow: Creates subshell
result=$(cat file.txt | grep "pattern" | wc -l)

# ✅ Fast: Direct processing
result=$(grep -c "pattern" file.txt)

# ❌ Slow: Multiple command substitutions
echo "User: $(whoami) Host: $(hostname) Time: $(date)"

# ✅ Fast: Fewer substitutions
user=$(whoami)
host=$(hostname)
echo "User: $user Host: $host Time: $(date)"
```

## **Part 4: Efficient Algorithms**

### **Optimal Patterns**

```bash
#!/bin/bash

# Efficient array processing
declare -a arr
for i in {1..1000}; do
    arr[$i]=$((i * 2))
done

# Fast string manipulation
str="long string here"
result="${str//old/new}"  # Parameter expansion, not sed

# Efficient number operations
result=$((10 + 20))  # Arithmetic expansion
# Not: result=$(echo "10 + 20" | bc)
```
