## **Part 1: Variable Basics**

### **Step 1: Creating and Using Variables**

```bash
#!/bin/bash

# Variable assignment (no spaces around =)
name="Khajanaraju"
age=30
city="Bangalore"

# Accessing variables
echo "Name: $name"
echo "Age: $age"
echo "City: $city"

# In double quotes - variables expand
echo "I am $name from $city"

# In single quotes - variables don't expand
echo 'I am $name from $city'  # Prints literally
```

### **Step 2: Variable Naming Rules**

```bash
# Valid variable names
var_name="value"
VAR_NAME="value"
_private="value"
var123="value"

# Invalid names (don't use)
# 123var="value"         # Starts with number
# var-name="value"       # Contains hyphen
# var name="value"       # Contains space
```

### **Step 3: Variable Assignment Methods**

```bash
#!/bin/bash

# Direct assignment
name="Khajanaraju"

# Assignment from command output
current_user=$(whoami)
date_now=$(date)

# Using backticks (older style, not recommended)
hostname=`hostname`

# Assignment with multiple lines
message="This is a \
multi-line \
string"

echo "User: $current_user"
echo "Date: $date_now"
```

## **Part 2: Variable Scope**

### **Global vs Local Variables**

```bash
#!/bin/bash

# Global variable (accessible everywhere)
global_var="I am global"

function test_scope() {
    # Local variable (only in function)
    local local_var="I am local"
    echo "Inside function: $local_var"
}

echo "Global: $global_var"
test_scope
# echo "Local: $local_var"  # Error - local_var not accessible here
```

### **Environment Variables**

```bash
#!/bin/bash

# Export variable to make it available to child processes
export MY_VAR="value"

# Built-in environment variables
echo "Home: $HOME"
echo "User: $USER"
echo "Shell: $SHELL"
echo "PATH: $PATH"
echo "PWD: $PWD"

# Create new environment variable
export DB_HOST="localhost"
export DB_PORT=5432
```

## **Part 3: Variable Expansion**

### **Parameter Expansion**

```bash
#!/bin/bash

# Simple expansion
name="Khajanaraju"
echo "Name: $name"

# Expansion with braces
echo "Name: ${name}"

# Default values
echo "Variable: ${undefined_var:-default_value}"

# Use default if empty
value=""
echo "Value: ${value:-empty}"

# Assign default if undefined
echo "New: ${new_var:=value}"
echo "New is now: $new_var"

# Display error if undefined
echo "Must exist: ${required:-Error: required not set}"
```

### **String Expansion**

```bash
#!/bin/bash

text="Hello World"

# Length of string
echo "Length: ${#text}"

# Extract substring
echo "Substring: ${text:0:5}"  # First 5 characters
echo "Substring: ${text:6}"    # From position 6 onwards

# Remove prefix
filename="file.txt"
echo "No extension: ${filename%.txt}"  # Remove .txt

# Remove suffix
path="/home/user/documents"
echo "No trailing: ${path%/}"
```

## **Part 4: String Manipulation**

### **Concatenation**

```bash
#!/bin/bash

first="Hello"
second="World"

# Concatenation (no operator needed)
result="$first $second"
echo "$result"

# Concatenation with braces
result="${first}${second}"
echo "$result"

# Building strings
message=""
message="${message}Line 1 "
message="${message}Line 2 "
message="${message}Line 3"
echo "$message"
```

### **Case Conversion**

```bash
#!/bin/bash

text="Hello World"

# Convert to uppercase
echo "${text^^}"

# Convert to lowercase
echo "${text,,}"

# Capitalize first letter
echo "${text^}"

# Replace first occurrence
echo "${text/World/Universe}"

# Replace all occurrences
text="apple apple apple"
echo "${text//apple/orange}"
```

## **Part 5: Special Variables**

### **Positional Parameters**

```bash
#!/bin/bash

# $0 = script name
# $1 = first argument
# $2 = second argument
# $@ = all arguments as array
# $* = all arguments as string
# $# = number of arguments

echo "Script: $0"
echo "First arg: $1"
echo "All args: $@"
echo "Total args: $#"

# Shifting arguments
echo "Before shift: $@"
shift
echo "After shift: $@"
```

### **Status Variables**

```bash
#!/bin/bash

# $? = exit status of last command
ls /tmp
if [ $? -eq 0 ]; then
    echo "Directory exists"
else
    echo "Directory not found"
fi

# $$ = process ID
echo "Script PID: $$"

# $! = last background process ID
sleep 10 &
echo "Background PID: $!"

# $- = current shell options
echo "Shell options: $-"
```

## **Part 6: Arrays**

### **Indexed Arrays**

```bash
#!/bin/bash

# Declare array
declare -a fruits

# Assign values
fruits[0]="apple"
fruits[1]="banana"
fruits[2]="orange"

# Access element
echo "First fruit: ${fruits[0]}"

# All elements
echo "All fruits: ${fruits[@]}"

# Array length
echo "Number of fruits: ${#fruits[@]}"

# Access by range
echo "First two: ${fruits[@]:0:2}"
```

### **Associative Arrays**

```bash
#!/bin/bash

# Declare associative array
declare -A student

# Assign values
student[name]="Khajanaraju"
student[age]="30"
student[city]="Bangalore"

# Access element
echo "Name: ${student[name]}"

# All keys and values
echo "All: ${student[@]}"

# Keys only
echo "Keys: ${!student[@]}"
```

## **Part 7: Read User Input**

### **Reading Input**

```bash
#!/bin/bash

# Simple read
read name
echo "You entered: $name"

# Read with prompt
read -p "Enter your name: " name
echo "Hello, $name"

# Read multiple values
read -p "Enter first and last name: " first last
echo "Full name: $first $last"

# Read into array
read -a arr -p "Enter values: "
echo "Array: ${arr[@]}"

# Read with timeout
read -t 5 -p "Enter quickly: " input
echo "You entered: $input"
```

## **Troubleshooting Guide**

| Issue | Solution |
|-------|----------|
| Variable not expanding | Use double quotes, not single |
| Variable undefined | Check spelling, initialize before use |
| Spaces in values | Quote the variable: `"$var"` |
| Array access error | Use correct syntax: `${array[index]}` |
| Command output issue | Use `$()` not backticks |

---

## **Best Practices**

1. **Quote variables** to prevent word splitting: `"$var"`
2. **Use braces for clarity** when needed: `${var}`
3. **Name variables clearly** - use descriptive names
4. **Initialize variables** before use
5. **Use local in functions** for function-scoped variables
6. **Export variables** if needed in child processes
7. **Validate input** from users
8. **Comment complex expansions** for clarity
9. **Use meaningful array indices** for clarity
10. **Avoid naming conflicts** with shell built-ins
