## **Part 1: EBS Volumes**

### **What is EBS?**

Elastic Block Storage (EBS) provides:
- Block-level storage volumes
- Persistent storage (survives instance termination)
- Network-attached (not physically attached)
- Automatic backups (snapshots)
- Encryption support

### **EBS Volume Types**

| Type | Use Case | Performance | Cost |
|------|----------|-------------|------|
| gp3 | General purpose | Good | Low |
| gp2 | General purpose | Baseline | Low |
| io1 | High IOPS (databases) | High | High |
| st1 | Throughput (big data) | High | Medium |
| sc1 | Cold storage | Low | Very Low |

### **Volume States**

```
creating → available → in-use → deleting → deleted
```

## **Part 2: Create and Manage EBS Volumes**

### **Create Volume**

1. Go to **EC2 → Volumes**
2. Click **Create volume**
3. Configure:
   ```
   Size: 20 GiB
   Availability Zone: Same as EC2 instance
   Volume Type: gp3
   IOPS: 3000
   Throughput: 125 MB/s
   Encrypted: Yes
   ```
4. Add tags: `Name: my-data-volume`
5. Click **Create volume**

### **Attach Volume to Instance**

1. Select volume
2. Click **Actions → Attach volume**
3. Select instance and device name (e.g., `/dev/sdf`)
4. Click **Attach**

### **Format and Mount Volume**

```bash
# Connect to instance
ssh -i my-key-pair.pem ubuntu@<ip>

# List available disks
lsblk

# Check if formatted
sudo file -s /dev/nvme1n1

# Format volume (if not formatted)
sudo mkfs -t ext4 /dev/nvme1n1

# Create mount point
sudo mkdir -p /data

# Mount volume
sudo mount /dev/nvme1n1 /data

# Verify mount
df -h

# Set permissions
sudo chmod 777 /data
```

### **Persistent Mount (Auto-mount on Reboot)**

```bash
# Get volume UUID
sudo blkid

# Edit fstab
sudo nano /etc/fstab

# Add line:
# UUID=xxxxx-xxxxx /data ext4 defaults 0 2

# Test mount
sudo mount -a

# Verify
mount | grep /data
```

### **Detach Volume**

1. Unmount on instance:
   ```bash
   sudo umount /data
   ```

2. In AWS Console:
   - Select volume
   - Click **Actions → Detach volume**
   - Volume returns to **available** state

## **Part 3: EBS Snapshots**

### **What are Snapshots?**

Snapshots are:
- Point-in-time backups of EBS volumes
- Stored in S3 (but managed by AWS)
- Can be used to create new volumes
- Can be shared across regions/accounts
- Incremental (only changes stored)

### **Create Snapshot**

```bash
# On instance, prepare data
sudo mkdir -p /data/important
echo "Critical data" | sudo tee /data/important/file.txt

# Flush file system
sudo sync
```

1. In AWS Console:
   - Go to **Volumes**
   - Select volume
   - Click **Actions → Create snapshot**
   - Description: `Backup of data volume`
   - Click **Create snapshot**

### **Monitor Snapshot Creation**

1. Go to **Snapshots**
2. Monitor progress (usually seconds to minutes)
3. Status: `pending → completed`

### **Create Volume from Snapshot**

1. Select snapshot
2. Click **Actions → Create volume from snapshot**
3. Configure:
   ```
   Size: >= Original size
   AZ: Same region as snapshot
   Volume Type: gp3
   Encrypted: Yes
   ```
4. Click **Create volume**
5. Attach to instance

### **Copy Snapshot to Another Region**

1. Select snapshot
2. Click **Actions → Copy snapshot**
3. Select destination region
4. Click **Copy**
5. Snapshot available in new region

### **Share Snapshot**

1. Select snapshot
2. Click **Modify permissions**
3. Choose:
   - **Private**: Only your account
   - **Public**: Anyone can use
   - **Specific AWS accounts**: Share with others
4. Click **Save**

### **Delete Snapshot**

1. Select snapshot
2. Click **Actions → Delete snapshot**
3. Confirm deletion
4. Note: Cannot delete if volume is in use

## **Part 4: Amazon Machine Images (AMI)**

### **What is an AMI?**

An AMI is:
- Template for launching instances
- Contains OS, software, configurations
- Can be custom-created
- Can be shared and copied
- Region-specific

### **Find Public AMIs**

1. In EC2 console: **AMIs**
2. Filter:
   - **Owner**: Amazon, Community, AWS Marketplace
   - **Root device type**: EBS, Instance Store
3. Popular: Ubuntu, Amazon Linux 2, CentOS

### **Create Custom AMI from Instance**

1. Launch instance and configure:
   ```bash
   sudo apt update
   sudo apt install -y apache2 nodejs
   sudo systemctl start apache2
   echo "<h1>Custom AMI</h1>" | sudo tee /var/www/html/index.html
   ```

2. In AWS Console:
   - Right-click instance
   - Click **Image and templates → Create image**
   - Enter:
     ```
     Image name: my-custom-ami
     Description: Apache + Node.js pre-installed
     ```
   - Click **Create image**

3. Monitor creation:
   - Go to **AMIs**
   - Status: `pending → available`

### **Launch Instance from Custom AMI**

1. Go to **AMIs**
2. Select custom AMI
3. Click **Launch**
4. Configuration:
   - Instance type, VPC, subnet, security group, key pair
   - Software already installed!
5. Connect and verify software present

### **Copy AMI to Another Region**

1. Select AMI
2. Click **Actions → Copy AMI**
3. Select destination region
4. Click **Copy AMI**
5. Available in new region

### **Share AMI with Others**

1. Select AMI
2. Click **Modify image permissions**
3. Choose sharing option
4. Add account IDs (for private sharing)
5. Click **Save**

### **Deregister AMI**

1. Select AMI
2. Click **Deregister**
3. Warning: Cannot be undone
4. Snapshots persist (delete separately if needed)

## **Part 6: Backup and Disaster Recovery**

### **Backup Strategy**

```
Regular Snapshots:
- Daily snapshots of critical volumes
- Weekly full backups
- Monthly archive backups

Lifecycle Management:
- Delete snapshots older than 30 days
- Keep monthly snapshots for 1 year
- Archive to S3 Glacier for long-term
```

### **Automated Snapshots with AWS Backup**

1. Go to **AWS Backup**
2. Create backup plan
3. Select resources and schedule
4. Define retention

## **Part 7: Performance Optimization**

### **EBS Optimization**

```bash
# Enable EBS optimization on instance
# (during launch or modification)

# Check I/O performance
iostat -x 1 5

# Monitor disk usage
df -h
du -sh /data
```

### **Volume Size Considerations**

```
- Larger volumes = better performance
- IOPS: 3,000 - 16,000 (gp3)
- Throughput: 125 - 1,000 MB/s (gp3)
- Pay per GB and per IOPS
```
