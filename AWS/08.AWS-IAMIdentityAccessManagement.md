## **Part 1: IAM Fundamentals**

IAM manages:
- Users (people)
- Groups (collections of users)
- Roles (for services/applications)
- Policies (permissions)
- Credentials (passwords, access keys)

---

## **Part 2: Create IAM Users**

```bash
# Via Console: IAM → Users → Create user
# Via CLI:
aws iam create-user --user-name john

# Create access key
aws iam create-access-key --user-name john

# Set login password
aws iam create-login-profile --user-name john --password MyPassword123! --password-reset-required

# List users
aws iam list-users
```

---

## **Part 3: Create Groups and Policies**

```bash
# Create group
aws iam create-group --group-name developers

# Create inline policy
aws iam put-group-policy \
  --group-name developers \
  --policy-name developer-policy \
  --policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Action": "ec2:*",
      "Resource": "*"
    }]
  }'

# Add user to group
aws iam add-user-to-group --group-name developers --user-name john

# List group members
aws iam get-group --group-name developers
```

---

## **Part 4: Managed Policies**

```bash
# Attach AWS managed policy
aws iam attach-user-policy \
  --user-name john \
  --policy-arn arn:aws:iam::aws:policy/ReadOnlyAccess

# Common managed policies:
# - AdministratorAccess
# - PowerUserAccess
# - ReadOnlyAccess
# - EC2FullAccess
# - S3FullAccess

# Attach to group
aws iam attach-group-policy \
  --group-name developers \
  --policy-arn arn:aws:iam::aws:policy/EC2FullAccess
```

---

## **Part 5: Roles for Services**

```bash
# Create role for EC2
aws iam create-role \
  --role-name ec2-app-role \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {
        "Service": "ec2.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }]
  }'

# Attach policy to role
aws iam attach-role-policy \
  --role-name ec2-app-role \
  --policy-arn arn:aws:iam::aws:policy/S3FullAccess

# Create instance profile
aws iam create-instance-profile --instance-profile-name ec2-profile

# Add role to instance profile
aws iam add-role-to-instance-profile \
  --instance-profile-name ec2-profile \
  --role-name ec2-app-role
```

## **Part 6: Best Practices**

```
1. Least Privilege: Give minimum permissions needed
2. Use groups: Manage permissions at group level
3. Enable MFA: Multi-factor authentication
4. Rotate credentials: Regularly change passwords/keys
5. No root access keys: Use root only when necessary
6. Audit access: Monitor CloudTrail logs
7. Use roles: For applications, not users
8. Policy conditions: Restrict by IP, time, etc.
```
