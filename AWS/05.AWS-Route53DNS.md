## **Part 1: Route53 Fundamentals**

### **What is Route53?**

Route53 provides:
- Domain registration
- DNS management
- Health checks
- Traffic policies
- Multi-region routing

### **DNS Record Types**

| Type | Purpose | Example |
|------|---------|---------|
| A | IPv4 address | example.com → 192.0.2.1 |
| AAAA | IPv6 address | example.com → 2001:0db8:: |
| CNAME | Canonical name | www.example.com → example.com |
| MX | Mail exchange | example.com → mail.example.com |
| TXT | Text records | SPF, DKIM records |
| NS | Name servers | Route53 nameservers |
| SOA | Start of authority | Zone information |

## **Part 2: Register Domain**

### **Register via Route53**

1. In Route53 console, click **Registered domains**
2. Click **Register domain**
3. Enter domain name (e.g., myawsdomain.com)
4. Check availability
5. Click **Add to cart**
6. Enter contact information
7. Complete payment
8. Wait for domain registration (24-48 hours)

### **Use Existing Domain**

1. If domain registered elsewhere, update nameservers
2. In Route53: **Hosted zones → Create hosted zone**
3. Enter domain name
4. Copy Route53 nameservers:
   ```
   ns-123.awsdns-45.com
   ns-678.awsdns-90.org
   ns-901.awsdns-23.net
   ns-234.awsdns-56.com
   ```
5. Add to your domain registrar's nameserver settings

## **Part 3: Hosted Zones and Records**

### **Create Hosted Zone**

```bash
# Via CLI
aws route53 create-hosted-zone \
  --name myawsdomain.com \
  --caller-reference $(date +%s)

# Get Zone ID
ZONE_ID=$(aws route53 list-hosted-zones-by-name --dns-name myawsdomain.com --query 'HostedZones[0].Id' --output text)
```

### **Create A Record**

```bash
# Point to EC2 instance
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "myawsdomain.com",
        "Type": "A",
        "TTL": 300,
        "ResourceRecords": [{"Value": "192.0.2.1"}]
      }
    }]
  }'
```

### **Create Subdomain (CNAME)**

```bash
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "www.myawsdomain.com",
        "Type": "CNAME",
        "TTL": 300,
        "ResourceRecords": [{"Value": "myawsdomain.com"}]
      }
    }]
  }'
```

### **Create MX Record (Email)**

```bash
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "myawsdomain.com",
        "Type": "MX",
        "TTL": 300,
        "ResourceRecords": [{"Value": "10 mail.myawsdomain.com"}]
      }
    }]
  }'
```

### **List Records**

```bash
# List all records in hosted zone
aws route53 list-resource-record-sets --hosted-zone-id $ZONE_ID

# Query specific record
nslookup myawsdomain.com
dig myawsdomain.com
```

## **Part 4: Alias Records**

### **Alias Record to AWS Resources**

```bash
# Alias to ELB
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "api.myawsdomain.com",
        "Type": "A",
        "AliasTarget": {
          "HostedZoneId": "Z35SXDOTRQ7X7K",
          "DNSName": "my-elb-1234567890.us-east-1.elb.amazonaws.com",
          "EvaluateTargetHealth": false
        }
      }
    }]
  }'
```

### **Benefits of Alias Records**

```
- No additional charges
- Auto health check
- Works with AWS resources
- Automatic updates if resource changes
```

## **Part 5: Routing Policies**

### **Simple Routing**

Default routing policy:
```bash
# Single resource, no special routing
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "simple.myawsdomain.com",
        "Type": "A",
        "TTL": 300,
        "ResourceRecords": [{"Value": "192.0.2.1"}]
      }
    }]
  }'
```

### **Weighted Routing**

Distribute traffic by percentage:
```bash
# Server 1: 70%
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "app.myawsdomain.com",
        "Type": "A",
        "TTL": 300,
        "SetIdentifier": "server-1",
        "Weight": 70,
        "ResourceRecords": [{"Value": "192.0.2.1"}]
      }
    }]
  }'

# Server 2: 30%
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "app.myawsdomain.com",
        "Type": "A",
        "TTL": 300,
        "SetIdentifier": "server-2",
        "Weight": 30,
        "ResourceRecords": [{"Value": "192.0.2.2"}]
      }
    }]
  }'
```

### **Latency-based Routing**

Route based on lowest latency:
```bash
# US region
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "app.myawsdomain.com",
        "Type": "A",
        "TTL": 300,
        "SetIdentifier": "us-east-1",
        "Region": "us-east-1",
        "ResourceRecords": [{"Value": "192.0.2.1"}]
      }
    }]
  }'

# EU region
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "app.myawsdomain.com",
        "Type": "A",
        "TTL": 300,
        "SetIdentifier": "eu-west-1",
        "Region": "eu-west-1",
        "ResourceRecords": [{"Value": "192.0.2.2"}]
      }
    }]
  }'
```

### **Failover Routing**

Active-passive failover:
```bash
# Primary (active)
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "app.myawsdomain.com",
        "Type": "A",
        "TTL": 300,
        "SetIdentifier": "primary",
        "Failover": "PRIMARY",
        "HealthCheckId": "health-check-id",
        "ResourceRecords": [{"Value": "192.0.2.1"}]
      }
    }]
  }'

# Secondary (standby)
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "app.myawsdomain.com",
        "Type": "A",
        "TTL": 300,
        "SetIdentifier": "secondary",
        "Failover": "SECONDARY",
        "ResourceRecords": [{"Value": "192.0.2.2"}]
      }
    }]
  }'
```

## **Part 6: Health Checks**

### **Create Health Check**

```bash
aws route53 create-health-check \
  --health-check-config '{
    "Type": "HTTP",
    "ResourcePath": "/health",
    "FullyQualifiedDomainName": "api.myawsdomain.com",
    "Port": 80,
    "RequestInterval": 30,
    "FailureThreshold": 3
  }'

HEALTH_CHECK_ID=$(aws route53 list-health-checks --query 'HealthChecks[0].Id' --output text)
```

### **Monitor Health Check**

```bash
# Get status
aws route53 get-health-check-status --health-check-id $HEALTH_CHECK_ID

# View health check details
aws route53 get-health-check --health-check-id $HEALTH_CHECK_ID
```
