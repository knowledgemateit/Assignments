## **Part 1: S3 Fundamentals**

### **What is S3?**

S3 (Simple Storage Service) provides:
- Object storage (not file system)
- Unlimited scalability
- High availability and durability
- Multiple storage classes
- Fine-grained access control

### **S3 Concepts**

```
Buckets: Top-level containers (like directories)
Objects: Files/data stored in buckets
Keys: Unique identifier for object (like path)
Regions: Geographic location of bucket

Bucket naming:
- 3-63 characters
- Lowercase letters, numbers, hyphens
- Globally unique (across all AWS accounts)
```

### **Storage Classes**

| Class | Use Case | Cost | Availability |
|-------|----------|------|--------------|
| S3 Standard | Frequent access | High | High |
| S3 Intelligent-Tiering | Variable access | Medium | High |
| S3 Standard-IA | Infrequent access | Low | High |
| S3 Glacier | Long-term archive | Very Low | Low |
| S3 Glacier Deep Archive | Rarely accessed | Lowest | Lowest |

## **Part 2: Create and Manage Buckets**

### **Create Bucket**

```bash
# Via CLI
aws s3 mb s3://my-unique-bucket-name --region us-east-1

# Via Console:
# S3 → Create bucket
# - Bucket name: my-unique-bucket-name
# - Region: us-east-1
# - Click Create
```

### **List Buckets**

```bash
# Via CLI
aws s3 ls

# Via Console: S3 → Buckets

# Output shows all buckets and creation date
```

### **Delete Bucket**

```bash
# Via CLI
aws s3 rb s3://my-unique-bucket-name

# Delete non-empty bucket
aws s3 rb s3://my-unique-bucket-name --force

# Note: Cannot delete bucket with objects (must delete objects first)
```

## **Part 3: Upload and Download Objects**

### **Upload Files**

```bash
# Single file
aws s3 cp myfile.txt s3://my-bucket/

# Upload with specific key
aws s3 cp myfile.txt s3://my-bucket/uploads/myfile.txt

# Upload entire directory
aws s3 cp ./local-dir/ s3://my-bucket/backup/ --recursive

# Upload with metadata
aws s3 cp myfile.txt s3://my-bucket/ --metadata "user=john,date=2024"

# Upload with storage class
aws s3 cp myfile.txt s3://my-bucket/ --storage-class STANDARD_IA
```

### **Download Files**

```bash
# Single file
aws s3 cp s3://my-bucket/myfile.txt ./

# Download to specific path
aws s3 cp s3://my-bucket/myfile.txt ./downloads/myfile.txt

# Download entire directory
aws s3 cp s3://my-bucket/backup/ ./local-backup/ --recursive

# Dry run (list files that would be downloaded)
aws s3 cp s3://my-bucket/ ./ --recursive --dryrun
```

### **Sync Buckets**

```bash
# Sync local to S3 (upload)
aws s3 sync ./local-dir/ s3://my-bucket/

# Sync S3 to local (download)
aws s3 sync s3://my-bucket/ ./local-dir/

# Sync with delete (remove files not in source)
aws s3 sync s3://source-bucket/ s3://dest-bucket/ --delete

# Sync with exclude patterns
aws s3 sync ./local-dir/ s3://my-bucket/ --exclude "*.tmp" --exclude ".git/*"
```

## **Part 4: Bucket Policies and ACLs**

### **Public Read Access**

```bash
# Create bucket policy for public read
aws s3api put-bucket-policy --bucket my-bucket --policy '{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": "*",
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::my-bucket/*"
  }]
}'

# Note: Enable public access settings first
aws s3api put-public-access-block \
  --bucket my-bucket \
  --public-access-block-configuration \
  "BlockPublicAcls=false,IgnorePublicAcls=false,BlockPublicPolicy=false,RestrictPublicBuckets=false"
```

### **Deny Unencrypted Uploads**

```bash
aws s3api put-bucket-policy --bucket my-bucket --policy '{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Deny",
    "Principal": "*",
    "Action": "s3:PutObject",
    "Resource": "arn:aws:s3:::my-bucket/*",
    "Condition": {
      "StringNotEquals": {
        "s3:x-amz-server-side-encryption": "AES256"
      }
    }
  }]
}'
```

### **Object ACL (Access Control List)**

```bash
# Make object public
aws s3api put-object-acl --bucket my-bucket --key myfile.txt --acl public-read

# Make object private
aws s3api put-object-acl --bucket my-bucket --key myfile.txt --acl private

# Grant permissions to specific user
aws s3api put-object-acl \
  --bucket my-bucket \
  --key myfile.txt \
  --grant-full-control id=user-canonical-id \
  --grant-read uri="http://acs.amazonaws.com/groups/global/AllUsers"
```

## **Part 5: Versioning and Lifecycle Policies**

### **Enable Versioning**

```bash
# Enable versioning
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled

# List object versions
aws s3api list-object-versions --bucket my-bucket

# Retrieve specific version
aws s3api get-object \
  --bucket my-bucket \
  --key myfile.txt \
  --version-id xxxxx \
  myfile-old.txt
```

### **Lifecycle Policies**

```bash
# Create lifecycle policy (transition to Glacier after 30 days)
aws s3api put-bucket-lifecycle-configuration \
  --bucket my-bucket \
  --lifecycle-configuration '{
  "Rules": [{
    "Id": "Archive old files",
    "Filter": {"Prefix": "logs/"},
    "Transitions": [{
      "Days": 30,
      "StorageClass": "GLACIER"
    }],
    "Expiration": {
      "Days": 365
    }
  }]
}'
```

## **Part 6: Static Website Hosting**

### **Enable Website Hosting**

```bash
# Configure bucket for website hosting
aws s3api put-bucket-website \
  --bucket my-bucket \
  --website-configuration '{
  "IndexDocument": {"Suffix": "index.html"},
  "ErrorDocument": {"Key": "error.html"}
}'

# Website URL: http://my-bucket.s3-website-us-east-1.amazonaws.com
```

### **Upload Website Files**

```bash
# Create HTML files
echo "<h1>Welcome!</h1>" > index.html
echo "<h1>404 Not Found</h1>" > error.html

# Upload files
aws s3 cp index.html s3://my-bucket/ --content-type text/html
aws s3 cp error.html s3://my-bucket/ --content-type text/html

# Make public (policy)
aws s3api put-object-acl --bucket my-bucket --key index.html --acl public-read
aws s3api put-object-acl --bucket my-bucket --key error.html --acl public-read
```

### **Custom Domain (Optional)**

```
1. Register domain (Route53 or other)
2. Create S3 bucket named: yourdomain.com
3. Enable website hosting
4. Create CNAME record pointing to S3 endpoint
5. Website accessible at http://yourdomain.com
```

## **Part 7: Security and Encryption**

### **Enable Default Encryption**

```bash
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
  "Rules": [{
    "ApplyServerSideEncryptionByDefault": {
      "SSEAlgorithm": "AES256"
    }
  }]
}'
```

### **Enable MFA Delete**

```bash
# Requires versioning enabled
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled,MFADelete=Enabled \
  --mfa "arn:aws:iam::account:user/username serial-number token"
```

### **Block Public Access**

```bash
# Block all public access
aws s3api put-public-access-block \
  --bucket my-bucket \
  --public-access-block-configuration \
  "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"
```
