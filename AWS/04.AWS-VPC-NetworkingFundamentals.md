## **Part 1: VPC Fundamentals**

### **What is a VPC?**

VPC (Virtual Private Cloud) provides:
- Isolated network environment
- Private IP space (RFC 1918 ranges)
- Subnets for resource organization
- Internet and VPN connectivity
- Network security controls

### **IP Addressing**

RFC 1918 Private Ranges:
```
10.0.0.0/8 (10.0.0.0 - 10.255.255.255)
172.16.0.0/12 (172.16.0.0 - 172.31.255.255)
192.168.0.0/16 (192.168.0.0 - 192.168.255.255)
```

## **Part 2: Create VPC**

### **Using AWS Console**

1. Go to **VPC → Your VPCs**
2. Click **Create VPC**
3. Configure:
   ```
   Name: my-vpc
   IPv4 CIDR: 10.0.0.0/16
   IPv6 CIDR: None (for now)
   Tenancy: Default
   ```
4. Click **Create VPC**

### **Using AWS CLI**

```bash
aws ec2 create-vpc --cidr-block 10.0.0.0/16 --tag-specifications 'ResourceType=vpc,Tags=[{Key=Name,Value=my-vpc}]'

# Get VPC ID
VPC_ID=$(aws ec2 describe-vpcs --filters "Name=tag:Name,Values=my-vpc" --query 'Vpcs[0].VpcId' --output text)
```

### **Enable DNS**

```bash
# In Console: VPC → VPC settings → Edit DNS hostname
# Set both:
# - DNS hostnames: Enable
# - DNS resolution: Enable

# Via CLI:
aws ec2 modify-vpc-attribute --vpc-id $VPC_ID --enable-dns-hostnames
aws ec2 modify-vpc-attribute --vpc-id $VPC_ID --enable-dns-support
```

## **Part 3: Create Subnets**

### **Public Subnet**

```bash
# Create public subnet
aws ec2 create-subnet \
  --vpc-id $VPC_ID \
  --cidr-block 10.0.1.0/24 \
  --availability-zone us-east-1a \
  --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=public-subnet-1}]'

PUBLIC_SUBNET_ID=$(aws ec2 describe-subnets --filters "Name=tag:Name,Values=public-subnet-1" --query 'Subnets[0].SubnetId' --output text)

# Enable auto-assign public IP
aws ec2 modify-subnet-attribute --subnet-id $PUBLIC_SUBNET_ID --map-public-ip-on-launch
```

### **Private Subnet**

```bash
# Create private subnet
aws ec2 create-subnet \
  --vpc-id $VPC_ID \
  --cidr-block 10.0.2.0/24 \
  --availability-zone us-east-1a \
  --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=private-subnet-1}]'

PRIVATE_SUBNET_ID=$(aws ec2 describe-subnets --filters "Name=tag:Name,Values=private-subnet-1" --query 'Subnets[0].SubnetId' --output text)
```

## **Part 4: Internet Gateway and Routing**

### **Create Internet Gateway**

```bash
# Create IGW
aws ec2 create-internet-gateway \
  --tag-specifications 'ResourceType=internet-gateway,Tags=[{Key=Name,Value=my-igw}]'

IGW_ID=$(aws ec2 describe-internet-gateways --filters "Name=tag:Name,Values=my-igw" --query 'InternetGateways[0].InternetGatewayId' --output text)

# Attach to VPC
aws ec2 attach-internet-gateway --internet-gateway-id $IGW_ID --vpc-id $VPC_ID
```

### **Create Route Table (Public)**

```bash
# Create route table
aws ec2 create-route-table \
  --vpc-id $VPC_ID \
  --tag-specifications 'ResourceType=route-table,Tags=[{Key=Name,Value=public-rt}]'

PUBLIC_RT_ID=$(aws ec2 describe-route-tables --filters "Name=tag:Name,Values=public-rt" --query 'RouteTables[0].RouteTableId' --output text)

# Add route to IGW (default route 0.0.0.0/0)
aws ec2 create-route \
  --route-table-id $PUBLIC_RT_ID \
  --destination-cidr-block 0.0.0.0/0 \
  --gateway-id $IGW_ID

# Associate with public subnet
aws ec2 associate-route-table \
  --subnet-id $PUBLIC_SUBNET_ID \
  --route-table-id $PUBLIC_RT_ID
```

### **Create Route Table (Private)**

```bash
# Create private route table
aws ec2 create-route-table \
  --vpc-id $VPC_ID \
  --tag-specifications 'ResourceType=route-table,Tags=[{Key=Name,Value=private-rt}]'

PRIVATE_RT_ID=$(aws ec2 describe-route-tables --filters "Name=tag:Name,Values=private-rt" --query 'RouteTables[0].RouteTableId' --output text)

# Associate with private subnet
aws ec2 associate-route-table \
  --subnet-id $PRIVATE_SUBNET_ID \
  --route-table-id $PRIVATE_RT_ID

# Note: Private instances need NAT gateway for outbound internet (optional)
```

## **Part 5: Security Groups and Network ACLs**

### **Create Security Group**

```bash
# Create security group
aws ec2 create-security-group \
  --group-name app-sg \
  --description "Application security group" \
  --vpc-id $VPC_ID

SG_ID=$(aws ec2 describe-security-groups --filters "Name=group-name,Values=app-sg" --query 'SecurityGroups[0].GroupId' --output text)

# Add inbound rules
aws ec2 authorize-security-group-ingress \
  --group-id $SG_ID \
  --protocol tcp \
  --port 80 \
  --cidr 0.0.0.0/0

aws ec2 authorize-security-group-ingress \
  --group-id $SG_ID \
  --protocol tcp \
  --port 443 \
  --cidr 0.0.0.0/0

aws ec2 authorize-security-group-ingress \
  --group-id $SG_ID \
  --protocol tcp \
  --port 22 \
  --cidr 10.0.0.0/16  # Only from VPC CIDR
```

### **Network Access Control Lists (NACLs)**

NACLs provide:
- Subnet-level firewall
- Stateless (must allow both directions)
- Lower priority than Security Groups

```bash
# List NACL for subnet
aws ec2 describe-network-acls --filters "Name=association.subnet-id,Values=$PUBLIC_SUBNET_ID"

# Create custom NACL (advanced topic)
aws ec2 create-network-acl --vpc-id $VPC_ID
```

## **Part 6: Launch Instances in VPC**

### **Launch in Public Subnet**

```bash
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --instance-type t3.micro \
  --key-name my-key-pair \
  --subnet-id $PUBLIC_SUBNET_ID \
  --security-group-ids $SG_ID \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=public-instance}]'
```

### **Launch in Private Subnet**

```bash
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --instance-type t3.micro \
  --key-name my-key-pair \
  --subnet-id $PRIVATE_SUBNET_ID \
  --security-group-ids $SG_ID \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=private-instance}]'
```

## **Part 7: NAT Gateway and Bastion Host**

### **NAT Gateway (For Private Instances)**

```bash
# Create Elastic IP
aws ec2 allocate-address --domain vpc

EIP_ID=$(aws ec2 describe-addresses --query 'Addresses[0].AllocationId' --output text)

# Create NAT Gateway in public subnet
aws ec2 create-nat-gateway \
  --subnet-id $PUBLIC_SUBNET_ID \
  --allocation-id $EIP_ID

NAT_ID=$(aws ec2 describe-nat-gateways --query 'NatGateways[0].NatGatewayId' --output text)

# Wait for NAT to be available
aws ec2 wait nat-gateway-available --nat-gateway-ids $NAT_ID

# Add route in private route table to NAT
aws ec2 create-route \
  --route-table-id $PRIVATE_RT_ID \
  --destination-cidr-block 0.0.0.0/0 \
  --nat-gateway-id $NAT_ID
```

### **Bastion Host (Jump Box)**

```bash
# Create security group for Bastion
aws ec2 create-security-group \
  --group-name bastion-sg \
  --description "Bastion host security group" \
  --vpc-id $VPC_ID

BASTION_SG=$(aws ec2 describe-security-groups --filters "Name=group-name,Values=bastion-sg" --query 'SecurityGroups[0].GroupId' --output text)

# Allow SSH from specific IP
aws ec2 authorize-security-group-ingress \
  --group-id $BASTION_SG \
  --protocol tcp \
  --port 22 \
  --cidr YOUR_IP/32

# Allow SSH to private instances
aws ec2 authorize-security-group-ingress \
  --group-id $SG_ID \
  --protocol tcp \
  --port 22 \
  --source-group $BASTION_SG

# Launch Bastion instance in public subnet
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --instance-type t3.micro \
  --key-name my-key-pair \
  --subnet-id $PUBLIC_SUBNET_ID \
  --security-group-ids $BASTION_SG \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=bastion-host}]'
```

## **Part 8: VPC Peering (Advanced)**

### **Peer Two VPCs**

```bash
# Create peering connection
aws ec2 create-vpc-peering-connection \
  --vpc-id $VPC_ID \
  --peer-vpc-id vpc-xxxxxxxx

# Accept peering connection
aws ec2 accept-vpc-peering-connection \
  --vpc-peering-connection-id pcx-xxxxxxxx

# Add route in route table
aws ec2 create-route \
  --route-table-id $PRIVATE_RT_ID \
  --destination-cidr-block 10.1.0.0/16 \
  --vpc-peering-connection-id pcx-xxxxxxxx
```
