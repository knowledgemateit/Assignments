## **Part 1: EC2 Fundamentals**

### **What is EC2?**

EC2 (Elastic Compute Cloud) provides resizable computing capacity in the cloud:
- On-demand virtual servers
- Pay as you go pricing
- Scalable infrastructure
- Global availability

### **Instance Types**

| Family | Use Case | Example |
|--------|----------|---------|
| General Purpose (T, M) | Web servers, small DBs | t3.micro, m5.large |
| Compute Optimized (C) | High-performance apps | c5.large |
| Memory Optimized (R, X) | Large databases, cache | r5.xlarge |
| Storage Optimized (I, H) | Data warehousing | i3.large |
| GPU Instances (P, G) | ML, graphics | p3.2xlarge |

### **Instance States**

```
pending → running → stopping → stopped → terminating → terminated
```

## **Part 2: AWS Console Setup**

### **Step 1: Create AWS Account**

1. Visit **https://aws.amazon.com**
2. Click **Create AWS Account**
3. Provide email, password, and billing information
4. Verify email and phone
5. Choose support plan (Free Tier eligible)

### **Step 2: Set Up AWS Console**

1. Sign in to **AWS Console**
2. Go to **Services → EC2**
3. Select a region (e.g., us-east-1, eu-west-1)
4. Note: Different regions have different availability

### **Step 3: Create Key Pair**

1. In EC2 dashboard, click **Key Pairs** (under Network & Security)
2. Click **Create key pair**
3. Enter name: `my-key-pair`
4. Format: **pem** (or ppk for PuTTY)
5. Download and save securely
6. Set permissions: `chmod 400 my-key-pair.pem`

---

## **Part 3: Launch EC2 Instance**

### **Step 1: Launch Instance**

1. In EC2 dashboard, click **Instances**
2. Click **Launch instances**

### **Step 2: Select AMI (Amazon Machine Image)**

1. Choose **Ubuntu Server 22.04 LTS** (or Amazon Linux 2)
2. Note the AMI ID

### **Step 3: Choose Instance Type**

1. Select **t3.micro** (Free Tier eligible)
2. Click **Next**

### **Step 4: Configure Instance Details**

```
Number of instances: 1
Network: VPC (default)
Subnet: Default
Auto-assign public IP: Enable
IAM role: (skip for now)
Monitoring: Basic
```

### **Step 5: Add Storage**

```
Volume type: gp3 or gp2
Size: 20 GB
Delete on termination: Yes
```

### **Step 6: Add Tags**

```
Key: Name
Value: my-first-instance

Key: Environment
Value: Development
```

### **Step 7: Configure Security Group**

1. Create new security group: `my-security-group`
2. Add rules:
   ```
   SSH (22) - Source: Your IP / 0.0.0.0/0
   HTTP (80) - Source: 0.0.0.0/0
   HTTPS (443) - Source: 0.0.0.0/0
   ```

### **Step 8: Review and Launch**

1. Review all settings
2. Select key pair: `my-key-pair`
3. Click **Launch instances**

### **Step 9: Verify Instance**

1. Instance state should show **running**
2. Note the **Public IP address**
3. Observe other details (Private IP, Instance ID, etc.)

## **Part 4: Connect to EC2 Instance**

### **From Linux/Mac (SSH)**

```bash
# Change key permissions
chmod 400 my-key-pair.pem

# Connect to instance
ssh -i my-key-pair.pem ubuntu@<public-ip>

# Or use Public DNS
ssh -i my-key-pair.pem ubuntu@ec2-xxx-xxx-xxx-xxx.compute.amazonaws.com

# Answer 'yes' to accept fingerprint
```

### **From Windows (Using PuTTY)**

1. Convert key: `my-key-pair.pem` to `my-key-pair.ppk` (PuTTY Key Gen)
2. Open PuTTY
3. Enter hostname: `ubuntu@<public-ip>`
4. Go to SSH → Auth
5. Select private key: `my-key-pair.ppk`
6. Click Open

### **From Windows (Using PowerShell/Git Bash)**

```powershell
# Ensure key has correct permissions
Set-ItemProperty -Path "my-key-pair.pem" -Name Attributes -Value "Archive"

# Connect
ssh -i my-key-pair.pem ubuntu@<public-ip>
```

## **Part 5: Install and Configure Software**

### **Update System**

```bash
sudo apt update
sudo apt upgrade -y
```

### **Install Web Server (Apache)**

```bash
sudo apt install -y apache2

# Start Apache
sudo systemctl start apache2
sudo systemctl enable apache2

# Verify running
sudo systemctl status apache2
```

### **Install Node.js**

```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
node --version
npm --version
```

### **Install Docker**

```bash
sudo apt install -y docker.io
sudo usermod -aG docker $USER
sudo systemctl start docker
docker --version
```

## **Part 7: Manage Instance Lifecycle**

### **Reboot Instance**

```bash
# From AWS Console: Right-click → Reboot

# From terminal:
sudo reboot

# Or from AWS CLI:
aws ec2 reboot-instances --instance-ids i-xxxxx --region us-east-1
```

### **Stop Instance**

```bash
# From AWS Console: Right-click → Instance State → Stop
# Instance is stopped but NOT terminated
# EBS volume persists
# Charges stop (except for EBS)
```

### **Start Instance**

```bash
# From AWS Console: Right-click → Instance State → Start
# Resumes from stopped state
# Public IP may change (unless Elastic IP assigned)
```

### **Terminate Instance**

```bash
# From AWS Console: Right-click → Instance State → Terminate
# WARNING: Instance and data are deleted
# This cannot be undone!
```

## **Part 8: Elastic IP Address**

### **Allocate Elastic IP**

1. Go to **Elastic IPs** (under Network & Security)
2. Click **Allocate Elastic IP address**
3. Region: Select your region
4. Click **Allocate**

### **Associate with Instance**

1. Select Elastic IP
2. Click **Associate address**
3. Select instance and network interface
4. Click **Associate**

### **Benefits**

```
- IP address persists across stop/start
- Can be reassigned to different instances
- For production use
```
