## **Part 1: Install Jenkins**

### **Option 1: Windows Installation**

1. Download Jenkins from **https://www.jenkins.io/download/**
2. Choose **Windows Installer (native)** and download
3. Run the installer:
   - Accept license agreement
   - Choose default installation path
   - Install as Windows service
   - Run installer as administrator

### **Option 2: Docker Installation (Recommended)**

1. Create a directory:
   ```powershell
   mkdir C:\jenkins-home
   ```

2. Run Jenkins container:
   ```powershell
   docker run -d -p 8080:8080 -p 50000:50000 `
     -v C:\jenkins-home:/var/jenkins_home `
     -v /var/run/docker.sock:/var/run/docker.sock `
     --name jenkins jenkins/jenkins:lts
   ```

3. Get initial admin password:
   ```powershell
   docker logs jenkins | findstr "initialAdminPassword"
   ```

### **Step 2: Access Jenkins**

1. Open browser: **http://localhost:8080**
2. Enter the initial admin password
3. Follow the setup wizard
4. Install suggested plugins
5. Create first admin user

## **Part 2: Configure Jenkins**

### **Step 1: Install Required Plugins**

1. Go to **Manage Jenkins → Plugin Manager**
2. Search and install these plugins:
   - **Pipeline**
   - **Git**
   - **GitHub Integration**
   - **Docker Pipeline**
   - **Maven Integration**
   - **Email Extension**

### **Step 2: Configure Credentials**

1. Go to **Manage Jenkins → Manage Credentials**
2. Add credentials:
   - **GitHub Personal Access Token**
   - **Docker Hub credentials**
   - **Git SSH key** (optional)

### **Step 3: Configure System**

1. Go to **Manage Jenkins → System Configuration**
2. Configure:
   - **GitHub API**
   - **Email settings**
   - **Maven installations**

## **Part 3: Create a Freestyle Job**

### **Step 1: Create New Job**

1. Click **New Item**
2. Enter job name: `MyFirstJob`
3. Select **Freestyle job**
4. Click **OK**

### **Step 2: Configure Source Code Management**

1. Under **Source Code Management**:
   - Select **Git**
   - Enter repository URL: `https://github.com/YourUsername/YourRepo.git`
   - Set branch: `*/main`

### **Step 3: Configure Build Trigger**

1. Under **Build Triggers**:
   - Check **GitHub hook trigger for GITScm polling**

### **Step 4: Add Build Steps**

1. Click **Add build step**
2. Select **Execute shell** (Linux/Mac) or **Execute Windows batch command** (Windows):
   ```bash
   echo "Building application..."
   echo "Build number: $BUILD_NUMBER"
   ```

### **Step 5: Add Post-Build Actions**

1. Click **Add post-build action**
2. Select **Archive the artifacts**:
   - Files to archive: `target/*.jar`

### **Step 6: Save and Build**

1. Click **Save**
2. Click **Build Now**
3. Check **Build Console Output**

## **Part 4: Create a Declarative Pipeline**

### **Step 1: Create Pipeline Job**

1. Click **New Item**
2. Enter name: `MyPipeline`
3. Select **Pipeline**
4. Click **OK**

### **Step 2: Define Pipeline**

1. Under **Pipeline**, select **Pipeline script**
2. Enter the following code:
   ```groovy
   pipeline {
       agent any
       
       environment {
           BUILD_VERSION = "1.0.${BUILD_NUMBER}"
           GIT_REPO = "https://github.com/YourUsername/YourRepo.git"
       }
       
       stages {
           stage('Checkout') {
               steps {
                   echo "Checking out code..."
                   git branch: 'main', url: "${GIT_REPO}"
               }
           }
           
           stage('Build') {
               steps {
                   echo "Building application..."
                   sh '''
                       echo "Build version: ${BUILD_VERSION}"
                       # Add your build commands here
                       echo "Build completed"
                   '''
               }
           }
           
           stage('Test') {
               steps {
                   echo "Running tests..."
                   sh '''
                       echo "Running unit tests"
                       # Add your test commands here
                   '''
               }
           }
           
           stage('Deploy') {
               steps {
                   echo "Deploying application..."
                   sh '''
                       echo "Deploy to production"
                       # Add your deploy commands here
                   '''
               }
           }
       }
       
       post {
           always {
               echo "Pipeline execution completed"
           }
           success {
               echo "Build succeeded!"
           }
           failure {
               echo "Build failed!"
           }
       }
   }
   ```

3. Click **Save**

### **Step 3: Run Pipeline**

1. Click **Build Now**
2. Monitor the pipeline stages
3. View console output

## **Part 5: Pipeline with Git Checkout**

### **Step 1: Update Pipeline Script**

1. Create `Jenkinsfile` in your Git repository:
   ```groovy
   pipeline {
       agent any
       
       options {
           timestamps()
           timeout(time: 1, unit: 'HOURS')
           buildDiscarder(logRotator(numToKeepStr: '10'))
       }
       
       stages {
           stage('Prepare') {
               steps {
                   script {
                       echo "Preparing build..."
                       sh 'git --version'
                       sh 'echo "Workspace: ${WORKSPACE}"'
                   }
               }
           }
           
           stage('Build') {
               steps {
                   script {
                       echo "Building..."
                       // For Maven projects
                       sh 'mvn clean package -DskipTests'
                       
                       // For Node.js projects
                       // sh 'npm install && npm run build'
                   }
               }
           }
           
           stage('Unit Tests') {
               steps {
                   script {
                       echo "Running unit tests..."
                       // sh 'mvn test'
                       // sh 'npm test'
                   }
               }
           }
           
           stage('Build Docker Image') {
               when {
                   branch 'main'
               }
               steps {
                   script {
                       echo "Building Docker image..."
                       sh '''
                           docker build -t myapp:${BUILD_NUMBER} .
                           docker tag myapp:${BUILD_NUMBER} myapp:latest
                       '''
                   }
               }
           }
           
           stage('Deploy to Dev') {
               when {
                   branch 'develop'
               }
               steps {
                   script {
                       echo "Deploying to development..."
                       // sh 'ansible-playbook deploy-dev.yml'
                   }
               }
           }
           
           stage('Deploy to Prod') {
               when {
                   branch 'main'
               }
               steps {
                   input 'Deploy to production?'
                   script {
                       echo "Deploying to production..."
                       // sh 'ansible-playbook deploy-prod.yml'
                   }
               }
           }
       }
       
       post {
           always {
               junit 'target/surefire-reports/*.xml'
               archiveArtifacts artifacts: 'target/*.jar', allowEmptyArchive: true
           }
           success {
               echo "Build and tests passed!"
           }
           failure {
               echo "Build or tests failed!"
               // emailext(
               //     subject: "Build Failed: ${env.JOB_NAME}",
               //     body: "Check console output at ${env.BUILD_URL}",
               //     to: "admin@example.com"
               // )
           }
       }
   }
   ```

## **Part 7: Advanced Pipeline Features**

### **Step 1: Create Parallel Stages**

1. Update your `Jenkinsfile`:
   ```groovy
   pipeline {
       agent any
       
       stages {
           stage('Build & Test') {
               parallel {
                   stage('Build') {
                       steps {
                           echo "Building..."
                           sh 'echo "Build step"'
                       }
                   }
                   stage('Lint') {
                       steps {
                           echo "Linting..."
                           sh 'echo "Lint step"'
                       }
                   }
                   stage('Security Scan') {
                       steps {
                           echo "Scanning..."
                           sh 'echo "Scan step"'
                       }
                   }
               }
           }
           
           stage('Deploy') {
               steps {
                   echo "Deploying..."
               }
           }
       }
   }
   ```

### **Step 2: Matrix Builds**

1. Add matrix builds for multiple configurations:
   ```groovy
   pipeline {
       agent any
       
       options {
           timestamps()
       }
       
       stages {
           stage('Matrix Build') {
               matrix {
                   agent any
                   axes {
                       axis {
                           name 'JAVA_VERSION'
                           values 'Java8', 'Java11', 'Java17'
                       }
                       axis {
                           name 'OS'
                           values 'Windows', 'Linux', 'macOS'
                       }
                   }
                   stages {
                       stage('Build') {
                           steps {
                               echo "Building with ${JAVA_VERSION} on ${OS}"
                           }
                       }
                   }
               }
           }
       }
   }
   ```

## **Part 8: Jenkins with Docker**

### **Docker Build in Pipeline**

1. Update your `Jenkinsfile`:
   ```groovy
   pipeline {
       agent any
       
       stages {
           stage('Build Docker Image') {
               steps {
                   script {
                       sh '''
                           docker build -t myapp:${BUILD_NUMBER} .
                       '''
                   }
               }
           }
           
           stage('Push to Docker Hub') {
               steps {
                   script {
                       withDockerRegistry(credentialsId: 'dockerhub-creds') {
                           sh '''
                               docker tag myapp:${BUILD_NUMBER} YourUsername/myapp:${BUILD_NUMBER}
                               docker push YourUsername/myapp:${BUILD_NUMBER}
                           '''
                       }
                   }
               }
           }
       }
   }
   ```
