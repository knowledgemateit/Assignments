## **Part 1: Install Kubernetes Locally**

### **Option 1: Using Docker Desktop (Recommended)**

1. Open **Docker Desktop Settings**
2. Go to **Kubernetes** tab
3. Check **Enable Kubernetes**
4. Click **Apply & Restart**
5. Wait for Kubernetes to start (5-10 minutes)

### **Option 2: Using Minikube**

1. Install Minikube from https://minikube.sigs.k8s.io/
2. Start Minikube:
   ```powershell
   minikube start
   ```

### **Step 2: Verify Kubernetes Installation**

1. Check cluster nodes:
   ```powershell
   kubectl get nodes
   ```
   **Expected Output:**
   ```
   NAME             STATUS   ROLES           AGE   VERSION
   docker-desktop   Ready    control-plane   5m    v1.27.0
   ```

2. Check cluster info:
   ```powershell
   kubectl cluster-info
   ```

## **Part 2: Understanding Kubernetes Concepts**

### **Key Concepts**

- **Pod**: Smallest deployable unit (one or more containers)
- **Deployment**: Manages replica pods
- **Service**: Exposes pods to network
- **ConfigMap**: Store configuration data
- **PersistentVolume**: Data storage
- **Namespace**: Virtual cluster for resource isolation

## **Part 3: Create Your First Pod**

### **Step 1: Create Pod Manifest**

1. Create a folder:
   ```powershell
   mkdir C:\K8sApp
   cd C:\K8sApp
   ```

2. Create `pod.yaml`:
   ```yaml
   apiVersion: v1
   kind: Pod
   metadata:
     name: nginx-pod
     labels:
       app: nginx
   spec:
     containers:
     - name: nginx-container
       image: nginx:latest
       ports:
       - containerPort: 80
   ```

### **Step 2: Create the Pod**

1. Apply the manifest:
   ```powershell
   kubectl apply -f pod.yaml
   ```
   **Expected Output:**
   ```
   pod/nginx-pod created
   ```

### **Step 3: Verify Pod Creation**

1. List pods:
   ```powershell
   kubectl get pods
   ```
   **Expected Output:**
   ```
   NAME        READY   STATUS    RESTARTS   AGE
   nginx-pod   1/1     Running   0          1m
   ```

### **Step 4: Get Pod Details**

1. Describe the pod:
   ```powershell
   kubectl describe pod nginx-pod
   ```

2. View pod logs:
   ```powershell
   kubectl logs nginx-pod
   ```

3. Execute command in pod:
   ```powershell
   kubectl exec -it nginx-pod -- /bin/bash
   ```

## **Part 4: Create a Deployment**

### **Step 1: Create Deployment Manifest**

1. Create `deployment.yaml`:
   ```yaml
   apiVersion: apps/v1
   kind: Deployment
   metadata:
     name: nginx-deployment
     labels:
       app: nginx
   spec:
     replicas: 3
     selector:
       matchLabels:
         app: nginx
     template:
       metadata:
         labels:
           app: nginx
       spec:
         containers:
         - name: nginx
           image: nginx:1.21
           ports:
           - containerPort: 80
           resources:
             requests:
               memory: "64Mi"
               cpu: "250m"
             limits:
               memory: "128Mi"
               cpu: "500m"
   ```

### **Step 2: Create the Deployment**

1. Apply the deployment:
   ```powershell
   kubectl apply -f deployment.yaml
   ```

2. Verify deployment:
   ```powershell
   kubectl get deployments
   ```
   **Expected Output:**
   ```
   NAME               READY   UP-TO-DATE   AVAILABLE   AGE
   nginx-deployment   3/3     3            3           2m
   ```

### **Step 3: View Replica Sets**

1. List ReplicaSets:
   ```powershell
   kubectl get replicasets
   ```

2. List all pods created by deployment:
   ```powershell
   kubectl get pods
   ```

## **Part 5: Create a Service**

### **Step 1: Create Service Manifest**

1. Create `service.yaml`:
   ```yaml
   apiVersion: v1
   kind: Service
   metadata:
     name: nginx-service
     labels:
       app: nginx
   spec:
     type: LoadBalancer
     selector:
       app: nginx
     ports:
     - protocol: TCP
       port: 80
       targetPort: 80
   ```

### **Step 2: Create the Service**

1. Apply the service:
   ```powershell
   kubectl apply -f service.yaml
   ```

2. View services:
   ```powershell
   kubectl get services
   ```
   **Expected Output:**
   ```
   NAME            TYPE           CLUSTER-IP      EXTERNAL-IP   PORT(S)
   nginx-service   LoadBalancer   10.96.0.1       localhost     80:30123/TCP
   ```

### **Step 3: Access the Service**

1. For Docker Desktop:
   ```
   http://localhost
   ```

2. For Minikube:
   ```powershell
   minikube service nginx-service
   ```

## **Part 6: ConfigMap and Environment Variables**

### **Step 1: Create ConfigMap**

1. Create `configmap.yaml`:
   ```yaml
   apiVersion: v1
   kind: ConfigMap
   metadata:
     name: app-config
   data:
     APP_ENV: production
     LOG_LEVEL: info
     DATABASE_HOST: postgres.default.svc.cluster.local
   ```

### **Step 2: Update Deployment to Use ConfigMap**

1. Create `deployment-with-config.yaml`:
   ```yaml
   apiVersion: apps/v1
   kind: Deployment
   metadata:
     name: app-deployment
   spec:
     replicas: 2
     selector:
       matchLabels:
         app: myapp
     template:
       metadata:
         labels:
           app: myapp
       spec:
         containers:
         - name: app
           image: myapp:1.0
           ports:
           - containerPort: 8080
           envFrom:
           - configMapRef:
               name: app-config
           env:
           - name: POD_NAME
             valueFrom:
               fieldRef:
                 fieldPath: metadata.name
   ```

### **Step 3: Apply ConfigMap and Deployment**

1. Apply ConfigMap:
   ```powershell
   kubectl apply -f configmap.yaml
   ```

2. Apply deployment:
   ```powershell
   kubectl apply -f deployment-with-config.yaml
   ```

## **Part 7: Scaling and Updates**

### **Scale a Deployment**

1. Scale to 5 replicas:
   ```powershell
   kubectl scale deployment nginx-deployment --replicas=5
   ```

2. Verify scaling:
   ```powershell
   kubectl get pods
   ```

### **Update Image in Deployment**

1. Update the image:
   ```powershell
   kubectl set image deployment/nginx-deployment nginx=nginx:1.23 --record
   ```

2. Check rollout status:
   ```powershell
   kubectl rollout status deployment/nginx-deployment
   ```

3. View rollout history:
   ```powershell
   kubectl rollout history deployment/nginx-deployment
   ```

### **Rollback to Previous Version**

1. Rollback:
   ```powershell
   kubectl rollout undo deployment/nginx-deployment
   ```

## **Part 8: Namespaces**

### **Create a Namespace**

1. Create namespace:
   ```powershell
   kubectl create namespace production
   ```

2. Apply resources to namespace:
   ```powershell
   kubectl apply -f deployment.yaml -n production
   ```

3. View resources in namespace:
   ```powershell
   kubectl get pods -n production
   ```

### **Delete Resources**

1. Delete deployment:

2. Delete service:
   ```powershell
   kubectl delete service nginx-service
   ```

3. Delete pod:
   ```powershell
   kubectl delete pod nginx-pod
   ```

4. Delete namespace:
   ```powershell
   kubectl delete namespace production
   ```
