### **Assignment: Nexus Artifact Repository - Manage Dependencies and Build Artifacts**

This assignment will teach you how to set up **Nexus Repository Manager**, manage artifacts, and configure Maven for dependency management.

## **Part 1: Install Nexus Repository Manager**

### **Option 1: Docker Installation (Recommended)**

1. Pull Nexus image:
   ```powershell
   docker pull sonatype/nexus3:latest
   ```

2. Create persistent volume:
   ```powershell
   docker volume create nexus-data
   ```

3. Run Nexus container:
   ```powershell
   docker run -d -p 8081:8081 `
     --name nexus `
     -v nexus-data:/nexus-data `
     sonatype/nexus3:latest
   ```

4. Wait for startup (2-3 minutes):
   ```powershell
   docker logs -f nexus
   ```

### **Option 2: Windows Installation**

1. Download from **https://www.sonatype.com/downloads/nexus-repository-oss**
2. Extract to `C:\nexus`
3. Navigate to `C:\nexus\bin`
4. Run `nexus.exe /run`

### **Step 2: Access Nexus**

1. Open browser: **http://localhost:8081**
2. Default login:
   - Username: **admin**
   - Password: Check `C:\nexus\sonatype-work\nexus3\admin.password` or container logs
3. Change password on first login

## **Part 2: Understand Nexus Repository Types**

Nexus supports different repository types:

- **Hosted**: Store your own artifacts
- **Proxy**: Cache remote repositories
- **Group**: Combine multiple repositories

---

## **Part 3: Create Repositories**

### **Step 1: Create Hosted Repository**

1. Click **Settings** → **Repositories**
2. Click **Create Repository**
3. Select **maven2 (hosted)**
4. Configure:
   - **Name**: `my-releases`
   - **Repository Realms**: Check **Maven**
   - **Deployment Policy**: **Allow redeploy**
5. Click **Create Repository**

### **Step 2: Create Snapshot Repository**

1. Repeat above for snapshots:
   - **Name**: `my-snapshots`
   - **Repository Realms**: Check **Maven**
   - **Version Policy**: **Snapshot**
   - **Deployment Policy**: **Allow redeploy**

### **Step 3: Create Proxy Repository**

1. Click **Create Repository**
2. Select **maven2 (proxy)**
3. Configure:
   - **Name**: `maven-central-proxy`
   - **Remote Storage**: `https://repo.maven.apache.org/maven2`
4. Click **Create Repository**

### **Step 4: Create Repository Group**

1. Click **Create Repository**
2. Select **maven2 (group)**
3. Configure:
   - **Name**: `maven-public`
   - **Member Repositories**: 
     - Add `my-releases`
     - Add `my-snapshots`
     - Add `maven-central-proxy`
4. Click **Create Repository**

## **Part 4: Configure Maven to Use Nexus**

### **Step 1: Update settings.xml**

1. Open `C:\Users\YourUsername\.m2\settings.xml` (create if doesn't exist):
   ```xml
   <settings>
       <servers>
           <server>
               <id>nexus-releases</id>
               <username>admin</username>
               <password>your-password</password>
           </server>
           <server>
               <id>nexus-snapshots</id>
               <username>admin</username>
               <password>your-password</password>
           </server>
       </servers>
       
       <mirrors>
           <mirror>
               <id>nexus</id>
               <mirrorOf>*</mirrorOf>
               <url>http://localhost:8081/repository/maven-public/</url>
           </mirror>
       </mirrors>
       
       <profiles>
           <profile>
               <id>nexus</id>
               <repositories>
                   <repository>
                       <id>nexus-releases</id>
                       <name>Nexus Releases</name>
                       <url>http://localhost:8081/repository/my-releases/</url>
                       <releases>
                           <enabled>true</enabled>
                       </releases>
                       <snapshots>
                           <enabled>false</enabled>
                       </snapshots>
                   </repository>
                   <repository>
                       <id>nexus-snapshots</id>
                       <name>Nexus Snapshots</name>
                       <url>http://localhost:8081/repository/my-snapshots/</url>
                       <releases>
                           <enabled>false</enabled>
                       </releases>
                       <snapshots>
                           <enabled>true</enabled>
                       </snapshots>
                   </repository>
               </repositories>
               <pluginRepositories>
                   <pluginRepository>
                       <id>nexus</id>
                       <name>Nexus Plugins</name>
                       <url>http://localhost:8081/repository/maven-public/</url>
                   </pluginRepository>
               </pluginRepositories>
           </profile>
       </profiles>
       
       <activeProfiles>
           <activeProfile>nexus</activeProfile>
       </activeProfiles>
   </settings>
   ```

### **Step 2: Test Maven Connection**

1. Test dependency download:
   ```powershell
   mvn dependency:get -Dartifact=junit:junit:4.13.2
   ```

2. Verify artifacts are cached in Nexus

## **Part 5: Deploy Artifacts to Nexus**

### **Step 1: Configure pom.xml**

1. Add distribution management to `pom.xml`:
   ```xml
   <distributionManagement>
       <repository>
           <id>nexus-releases</id>
           <url>http://localhost:8081/repository/my-releases/</url>
       </repository>
       <snapshotRepository>
           <id>nexus-snapshots</id>
           <url>http://localhost:8081/repository/my-snapshots/</url>
       </snapshotRepository>
   </distributionManagement>
   ```

### **Step 2: Deploy Release Artifact**

1. Update version to release:
   ```xml
   <version>1.0.0</version>
   ```

2. Deploy:
   ```powershell
   mvn clean deploy
   ```

   **Expected Output:**
   ```
   [INFO] Uploading to nexus-releases: 
   http://localhost:8081/repository/my-releases/...
   [INFO] BUILD SUCCESS
   ```

### **Step 3: Deploy Snapshot Artifact**

1. Update version to snapshot:
   ```xml
   <version>1.0.1-SNAPSHOT</version>
   ```

2. Deploy:
   ```powershell
   mvn clean deploy
   ```

### **Step 4: Verify in Nexus UI**

1. Go to **Browse** → **my-releases** or **my-snapshots**
2. Verify your artifacts are listed

## **Part 6: Search and Download Artifacts**

### **Step 1: Search Artifacts**

1. In Nexus, click **Search**
2. Search for your artifact
3. View:
   - Artifact details
   - Versions
   - Dependencies

### **Step 2: Download Artifact**

1. Click on artifact
2. Copy Maven coordinates:
   ```xml
   <dependency>
       <groupId>com.example</groupId>
       <artifactId>my-app</artifactId>
       <version>1.0.0</version>
   </dependency>
   ```

3. Add to your project's `pom.xml`

## **Part 7: Cleanup Policies**

### **Step 1: Configure Cleanup Policy**

1. Go to **Settings → Cleanup Policies**
2. Click **Create Cleanup Policy**
3. Configure:
   - **Name**: `old-snapshots`
   - **Criteria**: 
     - Last downloaded: `60 days ago`
     - Release type: `Snapshots`
   - **Actions**: `Delete`

### **Step 2: Assign to Repository**

1. Go to **Settings → Repositories**
2. Select repository
3. In **Cleanup Policy**: Select your policy
4. Save

## **Part 8: Security and Access Control**

### **Step 1: Create User Account**

1. Go to **Settings → Security → Users**
2. Click **Create Local User**
3. Configure:
   - **Username**: `developer`
   - **Password**: `dev-password`
   - **Email**: `dev@example.com`
   - **Roles**: `nx-developer`

### **Step 2: Create Custom Role**

1. Go to **Settings → Security → Roles**
2. Click **Create Role**
3. Configure:
   - **Role ID**: `my-developers`
   - **Role Name**: `My Developers`
   - **Privileges**: 
     - `nx-repository-view-*-*-read`
     - `nx-repository-view-*-*-browse`
     - `nx-repository-view-*-*-add`
     - `nx-repository-view-*-*-edit`

### **Step 3: Update settings.xml with New User**

1. Update `settings.xml`:
   ```xml
   <servers>
       <server>
           <id>nexus-releases</id>
           <username>developer</username>
           <password>dev-password</password>
       </server>
   </servers>
   ```
