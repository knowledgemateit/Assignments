## **Part 1: Install Maven**

### **Step 1: Download Maven**

1. Visit **https://maven.apache.org/download.cgi**
2. Download binary archive (apache-maven-3.9.x-bin.zip)

### **Step 2: Install Maven**

1. Extract to `C:\tools\maven`
2. Set JAVA_HOME environment variable:
   ```powershell
   [Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Java\jdk-17", "Machine")
   ```

3. Add Maven to PATH:
   ```powershell
   [Environment]::SetEnvironmentVariable("M2_HOME", "C:\tools\maven", "Machine")
   $env:Path += ";C:\tools\maven\bin"
   ```

### **Step 3: Verify Installation**

1. Open new PowerShell window:
   ```powershell
   mvn --version
   ```
   **Expected Output:**
   ```
   Apache Maven 3.9.0
   Maven home: C:\tools\maven
   Java version: 17.0.1
   ```

## **Part 2: Maven Project Structure**

### **Step 1: Create Project Directory**

1. Create a new folder:
   ```powershell
   mkdir C:\maven-projects
   cd C:\maven-projects
   ```

### **Step 2: Generate Project from Archetype**

1. Create a new Maven project:
   ```powershell
   mvn archetype:generate `
     -DgroupId=com.example `
     -DartifactId=my-app `
     -DarchetypeArtifactId=maven-archetype-quickstart `
     -DinteractiveMode=false
   ```

2. Navigate to project:
   ```powershell
   cd my-app
   ```

### **Step 3: Understand Project Structure**

Project structure should look like:
```
my-app/
├── pom.xml                 # Project configuration file
├── src/
│   ├── main/
│   │   ├── java/          # Source code
│   │   │   └── com/example/App.java
│   │   └── resources/     # Configuration files
│   └── test/
│       ├── java/          # Test code
│       │   └── com/example/AppTest.java
│       └── resources/
└── target/                # Build output
```

## **Part 3: Understanding pom.xml**

### **Step 1: Review pom.xml**

1. Open `pom.xml` in VS Code:
   ```powershell
   code pom.xml
   ```

2. The basic structure:
   ```xml
   <project>
       <modelVersion>4.0.0</modelVersion>
       
       <groupId>com.example</groupId>
       <artifactId>my-app</artifactId>
       <version>1.0-SNAPSHOT</version>
       <packaging>jar</packaging>
       
       <name>my-app</name>
       <description>My first Maven application</description>
       
       <properties>
           <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
           <maven.compiler.source>11</maven.compiler.source>
           <maven.compiler.target>11</maven.compiler.target>
       </properties>
       
       <dependencies>
           <dependency>
               <groupId>junit</groupId>
               <artifactId>junit</artifactId>
               <version>4.13.2</version>
               <scope>test</scope>
           </dependency>
       </dependencies>
   </project>
   ```

## **Part 4: Managing Dependencies**

### **Step 1: Add Dependencies**

1. Update `pom.xml` to add dependencies:
   ```xml
   <dependencies>
       <!-- Spring Framework -->
       <dependency>
           <groupId>org.springframework</groupId>
           <artifactId>spring-core</artifactId>
           <version>6.0.0</version>
       </dependency>
       
       <!-- Spring Web -->
       <dependency>
           <groupId>org.springframework</groupId>
           <artifactId>spring-web</artifactId>
           <version>6.0.0</version>
       </dependency>
       
       <!-- Logging -->
       <dependency>
           <groupId>org.slf4j</groupId>
           <artifactId>slf4j-api</artifactId>
           <version>2.0.0</version>
       </dependency>
       
       <!-- Testing -->
       <dependency>
           <groupId>junit</groupId>
           <artifactId>junit</artifactId>
           <version>4.13.2</version>
           <scope>test</scope>
       </dependency>
   </dependencies>
   ```

### **Step 2: Download Dependencies**

1. Download dependencies:
   ```powershell
   mvn dependency:download-sources
   ```

2. View dependency tree:
   ```powershell
   mvn dependency:tree
   ```

## **Part 5: Build Maven Project**

### **Step 1: Clean Build**

1. Clean previous builds:
   ```powershell
   mvn clean
   ```

### **Step 2: Compile Source Code**

1. Compile Java code:
   ```powershell
   mvn compile
   ```
   **Expected Output:**
   ```
   [INFO] Compiling 1 source file
   [INFO] BUILD SUCCESS
   ```

### **Step 3: Run Tests**

1. Execute unit tests:
   ```powershell
   mvn test
   ```

### **Step 4: Package Application**

1. Create JAR file:
   ```powershell
   mvn package
   ```
   **Expected Output:**
   ```
   [INFO] Building jar: C:\maven-projects\my-app\target\my-app-1.0-SNAPSHOT.jar
   [INFO] BUILD SUCCESS
   ```

### **Step 5: Full Build Lifecycle**

1. Run complete build:
   ```powershell
   mvn clean package
   ```

## **Part 6: Create Maven Web Application**

### **Step 1: Generate Web Project**

1. Create a web project:
   ```powershell
   mvn archetype:generate `
     -DgroupId=com.example `
     -DartifactId=my-web-app `
     -DarchetypeArtifactId=maven-archetype-webapp `
     -DinteractiveMode=false
   ```

### **Step 2: Update pom.xml for Web Project**

1. Open `pom.xml` in the new project:
   ```xml
   <project>
       <modelVersion>4.0.0</modelVersion>
       <groupId>com.example</groupId>
       <artifactId>my-web-app</artifactId>
       <version>1.0-SNAPSHOT</version>
       <packaging>war</packaging>
       
       <properties>
           <maven.compiler.source>11</maven.compiler.source>
           <maven.compiler.target>11</maven.compiler.target>
       </properties>
       
       <dependencies>
           <!-- Servlet API -->
           <dependency>
               <groupId>javax.servlet</groupId>
               <artifactId>javax.servlet-api</artifactId>
               <version>4.0.1</version>
               <scope>provided</scope>
           </dependency>
           
           <!-- JUnit -->
           <dependency>
               <groupId>junit</groupId>
               <artifactId>junit</artifactId>
               <version>4.13.2</version>
               <scope>test</scope>
           </dependency>
       </dependencies>
       
       <build>
           <plugins>
               <!-- Tomcat Maven Plugin -->
               <plugin>
                   <groupId>org.apache.tomcat.maven</groupId>
                   <artifactId>tomcat7-maven-plugin</artifactId>
                   <version>2.2</version>
                   <configuration>
                       <port>8080</port>
                       <path>/</path>
                   </configuration>
               </plugin>
           </plugins>
       </build>
   </project>
   ```

### **Step 3: Build Web Application**

1. Build WAR file:
   ```powershell
   mvn clean package
   ```

## **Part 7: Generate Reports**

### **Step 1: Add Reporting Plugins**

1. Update `pom.xml`:
   ```xml
   <reporting>
       <plugins>
           <!-- Surefire Report -->
           <plugin>
               <groupId>org.apache.maven.plugins</groupId>
               <artifactId>maven-surefire-report-plugin</artifactId>
               <version>3.0.0</version>
           </plugin>
           
           <!-- Code Coverage -->
           <plugin>
               <groupId>org.jacoco</groupId>
               <artifactId>jacoco-maven-plugin</artifactId>
               <version>0.8.8</version>
           </plugin>
           
           <!-- Javadoc -->
           <plugin>
               <groupId>org.apache.maven.plugins</groupId>
               <artifactId>maven-javadoc-plugin</artifactId>
               <version>3.4.1</version>
           </plugin>
       </plugins>
   </reporting>
   ```

### **Step 2: Generate Site Report**

1. Generate reports:
   ```powershell
   mvn site
   ```

2. View reports:
   ```
   target/site/index.html
   ```

## **Part 8: Maven Profiles**

### **Step 1: Create Profiles in pom.xml**

1. Update `pom.xml`:
   ```xml
   <profiles>
       <profile>
           <id>dev</id>
           <activation>
               <activeByDefault>true</activeByDefault>
           </activation>
           <properties>
               <environment>development</environment>
               <skipTests>true</skipTests>
           </properties>
       </profile>
       
       <profile>
           <id>test</id>
           <properties>
               <environment>test</environment>
               <skipTests>false</skipTests>
           </properties>
       </profile>
       
       <profile>
           <id>prod</id>
           <properties>
               <environment>production</environment>
               <skipTests>false</skipTests>
           </properties>
       </profile>
   </profiles>
   ```

### **Step 2: Use Profiles**

1. Build with specific profile:
   ```powershell
   mvn clean package -P prod
   ```

## **Part 9: Deploy Artifacts**

### **Step 1: Configure Maven Repository**

1. Create `settings.xml` (usually in `C:\Users\YourUsername\.m2`):
   ```xml
   <settings>
       <servers>
           <server>
               <id>nexus-repo</id>
               <username>admin</username>
               <password>password123</password>
           </server>
       </servers>
   </settings>
   ```

### **Step 2: Configure pom.xml for Distribution**

1. Update `pom.xml`:
   ```xml
   <distributionManagement>
       <repository>
           <id>nexus-repo</id>
           <url>http://localhost:8081/repository/releases/</url>
       </repository>
       
       <snapshotRepository>
           <id>nexus-repo</id>
           <url>http://localhost:8081/repository/snapshots/</url>
       </snapshotRepository>
   </distributionManagement>
   ```

### **Step 3: Deploy Artifact**

1. Deploy to repository:
   ```powershell
   mvn deploy
   ```
