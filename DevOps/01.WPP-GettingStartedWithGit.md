### **Assignment: Getting Started with Git - Version Control Basics**

This assignment will teach you the fundamentals of **Git**, including initialization, committing changes, branching, and collaboration workflows.

---

## **Objectives**

By completing this assignment, you will:
- ✅ Initialize a Git repository
- ✅ Understand staging and committing
- ✅ Work with branches
- ✅ Merge code changes
- ✅ Collaborate using Git workflows

---

## **Part 1: Initialize a Git Repository**

### **Step 1: Create a Project Directory**
1. Open **Command Prompt (cmd) or PowerShell**.
2. Create a new folder:
   ```powershell
   mkdir C:\MyFirstGitProject
   cd C:\MyFirstGitProject
   ```

### **Step 2: Initialize Git**
1. Run the following command to initialize Git:
   ```powershell
   git init
   ```
   **Expected Output:**
   ```
   Initialized empty Git repository in C:\MyFirstGitProject\.git/
   ```

### **Step 3: Verify Git Initialization**
1. List all files (including hidden ones):
   ```powershell
   dir -Force
   ```
   You should see a `.git` folder created.

---

## **Part 2: Configure Git User**

### **Step 1: Set Your Name and Email**
1. Configure your Git user details:
   ```powershell
   git config user.name "Your Name"
   git config user.email "your.email@example.com"
   ```

### **Step 2: Verify Configuration**
1. Check your configuration:
   ```powershell
   git config --global user.name
   git config --global user.email
   ```

---

## **Part 3: Create and Commit Your First File**

### **Step 1: Create a File**
1. Open **VS Code**:
   ```powershell
   code .
   ```

### **Step 2: Create a README.md File**
1. Create a new file named `README.md`.
2. Add the following content:
   ```markdown
   # My First Git Project

   This is my first Git repository.

   ## Features
   - Learning Git basics
   - Version control practice

   ## How to use
   1. Clone the repository
   2. Make changes
   3. Commit and push
   ```
3. Save the file.

### **Step 3: Check Git Status**
1. In the terminal, run:
   ```powershell
   git status
   ```
   **Expected Output:**
   ```
   On branch master
   No commits yet
   Untracked files:
     (use "git add <file>..." to include in what will be committed)
           README.md
   ```

### **Step 4: Stage the File**
1. Add the file to the staging area:
   ```powershell
   git add README.md
   ```

### **Step 5: Commit the File**
1. Commit with a message:
   ```powershell
   git commit -m "Initial commit: Add README.md"
   ```
   **Expected Output:**
   ```
   [master (root-commit) abc1234] Initial commit: Add README.md
    1 file changed, 10 insertions(+)
    create mode 100644 README.md
   ```

### **Step 6: View Commit History**
1. Check the commit log:
   ```powershell
   git log
   ```

## **Part 4: Working with Branches**

### **Step 1: Create a New Branch**
1. Create a new branch for a feature:
   ```powershell
   git branch feature/user-login
   ```

### **Step 2: List All Branches**
1. View all branches:
   ```powershell
   git branch -a
   ```
   **Expected Output:**
   ```
   * master
     feature/user-login
   ```

### **Step 3: Switch to the New Branch**
1. Checkout the feature branch:
   ```powershell
   git checkout feature/user-login
   ```
   or use the newer syntax:
   ```powershell
   git switch feature/user-login
   ```

### **Step 4: Make Changes in the New Branch**
1. Create a new file `user_login.py`:
   ```python
   # User Login Module
   
   def authenticate_user(username, password):
       """Authenticate user credentials"""
       # TODO: Implement authentication logic
       pass
   
   def logout_user():
       """Logout the current user"""
       # TODO: Implement logout logic
       pass
   ```
2. Save the file.

### **Step 5: Commit Changes in Feature Branch**
1. Stage and commit:
   ```powershell
   git add user_login.py
   git commit -m "Add user login module"
   ```

### **Step 6: Switch Back to Master**
1. Switch to master branch:
   ```powershell
   git checkout master
   ```
   Notice that `user_login.py` is not in the master branch.

---

## **Part 5: Merging Branches**

### **Step 1: Merge Feature Branch**
1. While on the master branch, merge the feature branch:
   ```powershell
   git merge feature/user-login
   ```
   **Expected Output:**
   ```
   Updating abc1234..def5678
   Fast-forward
    user_login.py | 10 insertions(+)
    1 file changed, 10 insertions(+)
   ```

### **Step 2: Verify Merge**
1. Check that the file now exists in master:
   ```powershell
   dir
   ```
   You should see both `README.md` and `user_login.py`.

### **Step 3: Delete the Feature Branch**
1. Delete the feature branch:
   ```powershell
   git branch -d feature/user-login
   ```

## **Part 6: Working with Remote Repositories**

### **Step 1: Connect to GitHub/GitLab**
1. Create a repository on **GitHub** (https://github.com) or **GitLab**.
2. Name it `MyFirstGitProject`.

### **Step 2: Add Remote Repository**
1. Add the remote URL:
   ```powershell
   git remote add origin https://github.com/YourUsername/MyFirstGitProject.git
   ```

### **Step 3: Push to Remote**
1. Push your commits to GitHub:
   ```powershell
   git push -u origin master
   ```

### **Step 4: Verify on GitHub**
1. Visit your GitHub repository in a browser.
2. Verify that your files are there.

---

## **Part 7: Cloning and Pulling Changes**

### **Step 1: Clone a Repository**
1. Create a new directory:
   ```powershell
   mkdir C:\MyFirstGitProject-Clone
   cd C:\MyFirstGitProject-Clone
   ```

2. Clone the repository:
   ```powershell
   git clone https://github.com/YourUsername/MyFirstGitProject.git
   ```

### **Step 2: Pull Latest Changes**
1. Make a change in the original repository (e.g., edit `README.md` on GitHub).
2. In the cloned directory, pull changes:
   ```powershell
   git pull origin master
   ```
