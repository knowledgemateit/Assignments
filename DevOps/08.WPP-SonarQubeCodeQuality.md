## **Part 1: Install SonarQube**

### **Option 1: Docker Installation (Recommended)**

1. Pull SonarQube image:
   ```powershell
   docker pull sonarqube:latest
   ```

2. Run SonarQube container:
   ```powershell
   docker run -d -p 9000:9000 `
     --name sonarqube `
     -e SONAR_JDBC_URL=jdbc:h2:./data/sonarqube_db `
     sonarqube:latest
   ```

3. Wait for startup (30-60 seconds)

### **Option 2: Windows Installation**

1. Download from **https://www.sonarqube.org/downloads/**
2. Extract to `C:\sonarqube`
3. Run `C:\sonarqube\bin\windows-x86-64\StartSonar.bat`

### **Step 2: Access SonarQube**

1. Open browser: **http://localhost:9000**
2. Default login:
   - Username: **admin**
   - Password: **admin**
3. Change password on first login

## **Part 2: Create SonarQube Project**

### **Step 1: Create New Project**

1. Click **Create Project**
2. Choose **Manual** project creation
3. Enter:
   - Project key: `my-app`
   - Display name: `My Application`
   - Click **Set Up**

### **Step 2: Generate Token**

1. You'll be prompted to select analysis method
2. Choose **Java** (or your language)
3. Click **Generate token**
4. Save the token (you'll need it)

### **Step 3: Select Build Tool**

1. Choose **Maven**
2. Copy the provided command

---

## **Part 3: Analyze Maven Project**

### **Step 1: Configure Maven Project**

1. Go to your Maven project directory:
   ```powershell
   cd C:\maven-projects\my-app
   ```

### **Step 2: Add SonarQube Plugin to pom.xml**

1. Open `pom.xml` and add in properties section:
   ```xml
   <properties>
       <sonar.projectKey>my-app</sonar.projectKey>
       <sonar.projectName>My Application</sonar.projectName>
       <sonar.host.url>http://localhost:9000</sonar.host.url>
       <sonar.sources>src/main/java</sonar.sources>
       <sonar.tests>src/test/java</sonar.tests>
   </properties>
   ```

### **Step 3: Run SonarQube Analysis**

1. Execute Maven analysis:
   ```powershell
   mvn clean verify sonar:sonar `
     -Dsonar.projectKey=my-app `
     -Dsonar.host.url=http://localhost:9000 `
     -Dsonar.login=your_generated_token
   ```

   **Expected Output:**
   ```
   [INFO] ANALYSIS SUCCESSFUL
   [INFO] You can now browse the results on: http://localhost:9000
   ```

### **Step 4: View Analysis Results**

1. Visit **http://localhost:9000**
2. Click on your project: **my-app**
3. Review:
   - Code smells
   - Bugs
   - Vulnerabilities
   - Coverage metrics
   - Duplications

## **Part 4: Code Quality Issues**

### **Step 1: Create Sample Code with Issues**

1. Create `C:\maven-projects\my-app\src\main\java\com\example\BadCode.java`:
   ```java
   package com.example;
   
   public class BadCode {
       
       // Issue: Unused variable
       int unusedVariable = 10;
       
       // Issue: Method too long
       public void processData(String input) {
           if (input != null) {
               if (input.length() > 0) {
                   if (input.contains("data")) {
                       System.out.println("Found data");
                       // Missing else clause
                   }
               }
           }
           // More code...
       }
       
       // Issue: Duplicate code
       public String duplicateMethod1() {
           String result = "";
           for (int i = 0; i < 10; i++) {
               result += "Item " + i + "\n";
           }
           return result;
       }
       
       public String duplicateMethod2() {
           String result = "";
           for (int i = 0; i < 10; i++) {
               result += "Item " + i + "\n";
           }
           return result;
       }
       
       // Issue: Empty catch block
       public void dangerousMethod() {
           try {
               int[] array = new int[10];
               array[20] = 5;
           } catch (ArrayIndexOutOfBoundsException e) {
               // Silently ignoring error
           }
       }
   }
   ```

### **Step 2: Re-run Analysis**

1. Run analysis again:
   ```powershell
   mvn clean verify sonar:sonar `
     -Dsonar.projectKey=my-app `
     -Dsonar.host.url=http://localhost:9000 `
     -Dsonar.login=your_token
   ```

2. View the issues found

## **Part 5: Fix Code Quality Issues**

### **Step 1: Review Issues in SonarQube**

1. In SonarQube dashboard, click **Issues**
2. Review each issue:
   - Code smell severity
   - Suggested fix
   - Affected code

### **Step 2: Fix the Issues**

1. Update `BadCode.java`:
   ```java
   package com.example;
   
   public class BadCode {
       
       // Fixed: Removed unused variable
       
       // Fixed: Refactored long method
       public void processData(String input) {
           if (input != null && !input.isEmpty()) {
               handleDataProcessing(input);
           }
       }
       
       private void handleDataProcessing(String input) {
           if (input.contains("data")) {
               System.out.println("Found data");
           } else {
               System.out.println("No data found");
           }
       }
       
       // Fixed: Extracted duplicate code to method
       public String generateItems() {
           StringBuilder result = new StringBuilder();
           for (int i = 0; i < 10; i++) {
               result.append("Item ").append(i).append("\n");
           }
           return result.toString();
       }
       
       public String duplicateMethod1() {
           return generateItems();
       }
       
       public String duplicateMethod2() {
           return generateItems();
       }
       
       // Fixed: Proper exception handling
       public void safeMethod() {
           try {
               int[] array = new int[10];
               array[20] = 5;
           } catch (ArrayIndexOutOfBoundsException e) {
               System.err.println("Array index out of bounds: " + e.getMessage());
               throw new RuntimeException("Invalid array access", e);
           }
       }
   }
   ```

### **Step 3: Verify Fixes**

1. Re-run analysis:
   ```powershell
   mvn clean verify sonar:sonar `
     -Dsonar.projectKey=my-app `
     -Dsonar.host.url=http://localhost:9000 `
     -Dsonar.login=your_token
   ```

2. Confirm issues are resolved

## **Part 6: Code Coverage**

### **Step 1: Add JaCoCo Plugin**

1. Update `pom.xml`:
   ```xml
   <build>
       <plugins>
           <plugin>
               <groupId>org.jacoco</groupId>
               <artifactId>jacoco-maven-plugin</artifactId>
               <version>0.8.8</version>
               <executions>
                   <execution>
                       <goals>
                           <goal>prepare-agent</goal>
                       </goals>
                   </execution>
                   <execution>
                       <id>report</id>
                       <phase>test</phase>
                       <goals>
                           <goal>report</goal>
                       </goals>
                   </execution>
               </executions>
           </plugin>
       </plugins>
   </build>
   ```

### **Step 2: Run Tests with Coverage**

1. Execute tests with coverage:
   ```powershell
   mvn clean test jacoco:report sonar:sonar `
     -Dsonar.projectKey=my-app `
     -Dsonar.host.url=http://localhost:9000 `
     -Dsonar.login=your_token
   ```

2. View coverage in SonarQube dashboard

## **Part 7: Quality Gates**

### **Step 1: Create Quality Gate**

1. In SonarQube, go to **Administration → Quality Gates**
2. Click **Create**
3. Name: `My Quality Gate`
4. Add conditions:
   - Code Coverage < 80% → FAIL
   - Duplicated Lines (%) > 5% → WARN
   - Issues (Blocker) > 0 → FAIL
   - Issues (Critical) > 5 → WARN

### **Step 2: Assign Quality Gate to Project**

1. Go to project settings
2. Select your quality gate
3. Save

### **Step 3: Verify Quality Gate**

1. Run analysis again
2. Check if project passes quality gate

## **Part 8: Jenkins Integration**

### **Step 1: Add SonarQube Plugin to Jenkins**

1. In Jenkins: **Manage Jenkins → Plugin Manager**
2. Search and install: **SonarQube Scanner**

### **Step 2: Configure Jenkins**

1. Go to **Manage Jenkins → Configure System**
2. Find **SonarQube servers** section
3. Click **Add**
4. Enter:
   - Name: `SonarQube`
   - Server URL: `http://localhost:9000`
   - Server authentication token: Your SonarQube token

### **Step 3: Update Jenkinsfile**

1. Create/update `Jenkinsfile`:
   ```groovy
   pipeline {
       agent any
       
       stages {
           stage('Checkout') {
               steps {
                   git branch: 'main', url: 'https://github.com/YourUsername/your-repo.git'
               }
           }
           
           stage('Build') {
               steps {
                   sh 'mvn clean package'
               }
           }
           
           stage('SonarQube Analysis') {
               steps {
                   script {
                       withSonarQubeEnv('SonarQube') {
                           sh 'mvn sonar:sonar'
                       }
                   }
               }
           }
           
           stage('Quality Gate') {
               steps {
                   script {
                       timeout(time: 1, unit: 'HOURS') {
                           waitForQualityGate abortPipeline: true
                       }
                   }
               }
           }
       }
   }
   ```
