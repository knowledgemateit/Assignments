### **Assignment: Docker Containerization - Build and Deploy Container Images**

This assignment will teach you how to **containerize an application** using Docker, create Docker images, and run containers.

---

## **Objectives**

By completing this assignment, you will:
- ✅ Understand Docker concepts (images, containers, registries)
- ✅ Install Docker on Windows
- ✅ Create a Dockerfile
- ✅ Build and run Docker containers
- ✅ Push images to Docker Hub

---

## **Part 1: Install Docker Desktop on Windows**

### **Step 1: Download Docker Desktop**
1. Visit **https://www.docker.com/products/docker-desktop**
2. Click **"Download for Windows"**
3. Run the installer and follow the installation wizard
4. Enable **WSL 2** (Windows Subsystem for Linux 2) when prompted
5. Restart your computer

### **Step 2: Verify Docker Installation**
1. Open **PowerShell** or **Command Prompt**
2. Check Docker version:
   ```powershell
   docker --version
   ```
   **Expected Output:**
   ```
   Docker version 24.0.0, build abc1234
   ```

3. Run a test container:
   ```powershell
   docker run hello-world
   ```
   **Expected Output:**
   ```
   Hello from Docker!
   ...
   ```

## **Part 2: Create a Simple Application**

### **Step 1: Create Project Directory**
1. Create a new folder:
   ```powershell
   mkdir C:\DockerApp
   cd C:\DockerApp
   ```

### **Step 2: Create a Python Application**
1. Create a file named `app.py`:
   ```python
   from flask import Flask
   
   app = Flask(__name__)
   
   @app.route('/')
   def hello():
       return 'Hello from Docker! 🐳'
   
   @app.route('/api/health')
   def health():
       return {'status': 'healthy'}
   
   if __name__ == '__main__':
       app.run(host='0.0.0.0', port=5000, debug=True)
   ```

### **Step 3: Create Requirements File**
1. Create `requirements.txt`:
   ```
   Flask==2.3.0
   Werkzeug==2.3.0
   ```

### **Step 4: Create a Dockerfile**
1. Create a file named `Dockerfile` (no extension):
   ```dockerfile
   # Use official Python runtime as base image
   FROM python:3.11-slim
   
   # Set working directory in container
   WORKDIR /app
   
   # Copy requirements file
   COPY requirements.txt .
   
   # Install Python dependencies
   RUN pip install --no-cache-dir -r requirements.txt
   
   # Copy application code
   COPY app.py .
   
   # Expose port 5000
   EXPOSE 5000
   
   # Set environment variables
   ENV FLASK_APP=app.py
   ENV FLASK_ENV=production
   
   # Run the application
   CMD ["python", "app.py"]
   ```

### **Step 5: Create .dockerignore File**
1. Create `.dockerignore`:
   ```
   __pycache__
   *.pyc
   .git
   .gitignore
   .env
   ```

## **Part 3: Build a Docker Image**

### **Step 1: Build the Docker Image**
1. In the `C:\DockerApp` directory, run:
   ```powershell
   docker build -t my-flask-app:1.0 .
   ```
   
   **Explanation:**
   - `-t`: Tag the image with a name and version
   - `.`: Look for Dockerfile in the current directory

   **Expected Output:**
   ```
   Sending build context to Docker daemon  2.048kB
   Step 1/8 : FROM python:3.11-slim
   ...
   Successfully built abc1234def56
   Successfully tagged my-flask-app:1.0
   ```

### **Step 2: List Docker Images**
1. View all images on your system:
   ```powershell
   docker images
   ```
   **Expected Output:**
   ```
   REPOSITORY       TAG       IMAGE ID       CREATED         SIZE
   my-flask-app     1.0       abc1234def56   2 minutes ago    156MB
   python           3.11-slim abc1234def56   2 weeks ago      125MB
   hello-world      latest    abc1234def56   1 month ago      13.3kB
   ```

## **Part 4: Run a Docker Container**

### **Step 1: Run the Container**
1. Run your Docker image as a container:
   ```powershell
   docker run -p 5000:5000 --name my-flask-container my-flask-app:1.0
   ```
   
   **Explanation:**
   - `-p 5000:5000`: Map port 5000 on host to port 5000 in container
   - `--name`: Give the container a name
   - Last argument: Image name and tag

### **Step 2: Test the Application**
1. Open a web browser and visit:
   ```
   http://localhost:5000
   ```
   Expected: See "Hello from Docker! 🐳"

2. Visit the health endpoint:
   ```
   http://localhost:5000/api/health
   ```
   Expected: See `{"status": "healthy"}`

### **Step 3: View Running Containers**
1. Open a **new** PowerShell window and run:
   ```powershell
   docker ps
   ```
   **Expected Output:**
   ```
   CONTAINER ID   IMAGE              COMMAND          STATUS           PORTS
   abc1234def56   my-flask-app:1.0   "python app.py"  Up 2 minutes     0.0.0.0:5000->5000/tcp
   ```

### **Step 4: Stop the Container**
1. In the new PowerShell window, stop the container:
   ```powershell
   docker stop my-flask-container
   ```

### **Step 5: Remove the Container**
1. Remove the container:
   ```powershell
   docker rm my-flask-container
   ```

## **Part 5: Container Management**

### **View Container Logs**
1. Run container in detached mode:
   ```powershell
   docker run -d -p 5000:5000 --name my-app my-flask-app:1.0
   ```

2. View logs:
   ```powershell
   docker logs my-app
   ```

3. Follow logs in real-time:
   ```powershell
   docker logs -f my-app
   ```

### **Execute Commands in Running Container**
1. Open an interactive shell:
   ```powershell
   docker exec -it my-app /bin/bash
   ```

### **Inspect Container Details**
1. Get detailed information:
   ```powershell
   docker inspect my-app
   ```

## **Part 6: Push Image to Docker Hub**

### **Step 1: Create Docker Hub Account**
1. Visit **https://hub.docker.com** and create an account
2. Verify your email

### **Step 2: Login to Docker Hub**
1. Login from command line:
   ```powershell
   docker login
   ```
   Enter your Docker Hub username and password

### **Step 3: Tag Image for Docker Hub**
1. Tag your image:
   ```powershell
   docker tag my-flask-app:1.0 YourDockerHubUsername/my-flask-app:1.0
   ```

### **Step 4: Push to Docker Hub**
1. Push the image:
   ```powershell
   docker push YourDockerHubUsername/my-flask-app:1.0
   ```

### **Step 5: Verify on Docker Hub**
1. Visit **https://hub.docker.com** and check your repositories

## **Part 7: Docker Compose (Bonus)**

### **Step 1: Create docker-compose.yml**
1. Create a file named `docker-compose.yml`:
   ```yaml
   version: '3.8'
   
   services:
     web:
       build: .
       ports:
         - "5000:5000"
       container_name: my-flask-app
       environment:
         - FLASK_ENV=production
       volumes:
         - .:/app
   ```

### **Step 2: Run with Docker Compose**
1. Start containers:
   ```powershell
   docker-compose up
   ```

2. Stop containers:
   ```powershell
   docker-compose down
   ```
